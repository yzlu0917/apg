## Blockers And Decisions

- Human blind audit still requires a reviewer outside the current agent loop, but external review should now wait for the `LLM-assisted benchmark v2` branch to harden algebra and graph/path further.
- Raw prompting is sufficient for diagnosis but too weak for AMCD; trained frozen heads are a much stronger reference point.
- The original pilot was too easy; the harder slice is more informative, especially on graph/path.
- Additional paraphrase diversity alone is unlikely to be enough, because the masked baseline survives held-out verbalizer evaluation.
- The strongest current distinction is between prompt evaluators and trained evaluators, not yet between visible and masked trained heads.
- External judge behavior is now consistent with that distinction, so the next highest-value test is realism rather than yet another synthetic baseline.
- Naturalized transfer is already enough to raise `ASS_total` for trained masked heads, so moving toward more realistic traces is likely to be more informative than immediately adding repair losses.
- Model-generated traces reinforce the judge-vs-trained distinction, but still lack audited local labels; if we need stronger process claims on natural data, annotation or executable alignment is the next bottleneck.
- The new auditable-generated slice resolves the `AMCD` bottleneck only partially, because coverage is now heavily skewed toward algebra.
- Graph/path responds to stronger output-format constraints, but blocksworld does not; planning-style natural generation appears to require more than prompt-only tightening at this model scale.
- With Ark credentials restored, the external judge on the hybrid slice again behaves more like a judge-style evaluator than like the trained verifier.
- Blocksworld can be recovered with a legality-aware scaffold, but the resulting slice is harder enough that trained `AMCD` drops materially; the main effect is harder process discrimination rather than a new burst of trained answer sensitivity.
- The repair pilot has now changed the decision surface: minimal repair is no longer purely speculative because `cond-swap` clearly improves `AMCD` on the hardest current OOD slice, but it does so with a real ordinary-AUROC trade-off.
- Joint repair is not automatically better than a single constraint. On the current setup it wins on synthetic validation and loses on generated OOD transfer, so scaling the most complicated loss by default would be premature.
- A light natural-language shift is enough to erase the `cond-swap` `AMCD` win while preserving its low-`ASS_total` advantage. The current evidence fits a faithfulness-utility Pareto frontier more than a universal repair win.
- The same naturalized slice also makes the masked baseline answer-sensitive again, so surface shift itself is now a first-class stressor rather than a cosmetic perturbation.
- Self-mined hard-negative replay currently pushes the model back toward utility at the cost of answer sensitivity. In the current form it does not dominate the frontier, so it should not be promoted as the default Week 3 repair path.
- Fixing replay coverage alone is not enough. Even domain-balanced and blocksworld-focused replay fail to produce a simple dominant method on the naturalized full-hybrid slice.
- The mechanism story is now stronger than the algorithm story. Even without a single dominating repair variant, the local `ASS -> AMCD -> utility` chain is supported on the audited OOD setup.
- Explicit factorization is not automatically enough either. In the current frozen-backbone pilot, the minimal dual-head does not use its consistency channel at optimum and therefore does not tighten the frontier.
- A more faithful local-process implementation does not rescue the current dual-head story. If even a step-scan `G_proc` still drives `alpha` to zero, then the current bottleneck is not just architectural wording; it is model/data scale or signal quality.
- Week 4 changes the deployment picture more than the mechanism picture. A stronger reranker can beat the frozen heads on utility, but it still works best in the masked condition, so scale alone does not remove answer leakage risk.
- Calibration now looks like a separate downstream problem. On the current reranker pilots, ranking and exploitability improve much more than `ECE / Brier`, which matches the proposal's decision to demote the calibration layer from the core method.
- Week 5 now shows that the masked reranker advantage survives cross-family attack transfer, not only average-case evaluation. The current best deployment target is therefore `Qwen3-Reranker-8B` masked, not a visible reranker and not the current frozen-head repair.
- Failure slices are now concrete enough to guide Week 6: planning-style naturalized blocksworld is the main weak group; verbalizer-family sensitivity is more problematic for visible reranking than masked reranking.
- Week 6 shows the current main comparison is no longer fragile to one seed. The strongest deployment conclusion now rests on average-case, worst-group, attacker-transfer, and bootstrap evidence at once.
- `q2h` shows that benchmark-v2 coverage recovery and blind-facing quality are now partially decoupled. Fixing algebra full-trace failures can improve rewritten counts without improving the blind packet and can reintroduce shallow length leakage elsewhere.
- `q2i` adds a second constraint: making algebra source semantics more subtle can itself break rewrite robustness badly enough to destroy blind-packet eligibility. Source-level realism cannot be pushed independently of rewrite compatibility.
- `q2j` adds the third constraint: maximizing rewrite faithfulness can directly hurt blind audit by making local arithmetic inconsistencies more legible to human reviewers.
- The next benchmark phase should not be another `q2*` patch cycle. It should be treated as a separate `benchmark-v3` project with stronger generation and reviewer-aware filtering.
- The first `benchmark-v3` strict smoke already shows a new selector failure mode: minimizing invalid-pick asymmetry is not enough if the selected family still contains a clearly synthetic valid trace.
- The hard-gated rerun shows a second constraint: once the selector is made truly strict, the current `k=2` candidate pool clears nothing at all. The next bottleneck is upstream candidate quality or reviewer cascade design, not gate implementation.
- Pairwise cascade sharpens that conclusion further: current candidate quality is not purely a single-trace problem. Some candidates look acceptable in relative within-role comparisons, but the resulting valid/invalid pairs still look patched when compared side by side.
- Pair-conditioned generation sharpens it one more step: even when jointly generated pairs survive candidate pruning cleanly, the pairwise reviewer still finds the final valid/invalid contrast too detectable. The next step has to attack contrast formation itself.
- Pair-level adversarial prune makes the current bottleneck even sharper: preserving joint pairs is correct, but still insufficient. The remaining failure is no longer an artifact of independent role pruning.
- Contrast-aware pair generation sharpens it one more time: even when the prompt explicitly optimizes balanced side-by-side naturalness, the final strict pair gate still rejects every smoke quartet. The next blocker is likely model/reviewer strength, not prompt wording.
- Reviewer-in-the-loop filtering must hide answer identity. If reviewer selection sees swapped-answer contradictions, it optimizes the wrong object and the strict benchmark line is polluted at construction time.
- The current full Cartesian search is already expensive at smoke scale. With `2` candidates per role and `3` quartets, the strict smoke consumed `216` API calls, so larger runs need pruning or cascaded filtering.
