## 1. 方向地图（含主题覆盖｜Step A：广撒网）

**时间窗口说明**：下面的“近期热门/有影响力”以 **2024-09 ～ 2026-03（最近 6–18 个月）** 的 arXiv / OpenReview / 顶会论文与基准为主（必要时补充少量更早但仍是“范式基座”的工作）。

### 1.1 推理与验证（reasoning / verifier / self-correction）

* 主线：用 **verifier / PRM（Process Reward Model）** 做 test-time scaling（Best-of-N、树搜索、rollout 评价），并把“验证”从结果验算推进到“过程监督/过程奖励”。([arXiv][1])
* 痛点：**reward hacking / verifier exploit**、标注稀缺、跨域泛化差（数学→代码/工具）、以及“越算越错”的非单调现象。([OpenReview][2])
* 重要性：这是目前最像“用机制提升推理”的路线之一：能把改进归因到 **验证信号如何塑形搜索/优化**。([arXiv][1])

### 1.2 可靠性与对齐（robustness / honesty / calibration / uncertainty）

* 主线：把 LLM 输出当作“可拒答/可选择”的决策系统：**不确定性估计、选择性预测、conformal 保证**、以及面向高风险场景的 calibration。([NeurIPS][3])
* 痛点：calibration 常在 **域迁移、长尾问题、推理链条依赖** 下失效；“准确≠可靠不确定性”的反直觉现象频繁出现。([NeurIPS][4])
* 重要性：**可证伪、可复现**（coverage / risk）很强；容易形成 best-paper 级“规范化评测协议”。([arXiv][5])

### 1.3 表征与机制解释（mechanistic interpretability / representation）

* 主线：Sparse Autoencoders（SAE）把激活分解为稀疏特征，做 **特征发现、单语义性、可控 steering** 与 scaling 行为研究。([OpenReview][6])
* 痛点：SAE 的“特征是否稳定/可迁移/可因果解释”仍争议很大；训练成本与 dead latents 等工程问题会拖累研究闭环。([OpenReview][6])
* 重要性：一旦把 SAE 特征与“错误/欺骗/幻觉”建立因果联系，极易形成 best-paper 叙事（机制+干预）。([OpenReview][7])

### 1.4 训练与优化新范式（RLHF/RLAIF alternatives / off-policy / direct preference）

* 主线：从 RLHF 走向 **离线偏好优化**（ORPO / SimPO / KTO / SimPER 等），强调“**少超参、无 reference model、更稳**”。([arXiv][8])
* 痛点：赛道拥挤、增量改动容易“工程化”；关键缺口转向 **原理解释（为什么这些 loss 有效）** 与跨任务统一。([arXiv][9])
* 重要性：对算力友好；若能提出统一理论/新评测维度，仍有 best-paper 空间。([arXiv][9])

### 1.5 数据合成与评测（synthetic data / evaluation / contamination / benchmark design）

* 主线：更真实的任务与更可靠评测：SWE-bench 这类“真实软件问题”、以及 **污染检测/污染影响因果量化/动态无污染评测**。([ICLR 会议录][10])
* 痛点：benchmark 被“刷分”、数据泄漏、统计显著性不足；以及“模型改进”与“评测变动”纠缠不清。([arXiv][11])
* 重要性：这是 **最容易形成可复现、可验证贡献** 的领域之一（新 benchmark / 新 protocol / 新风险度量）。([ACL Anthology][12])

### 1.6 高效学习（PEFT / low-data / modularity / compositionality）

* 主线：用 LoRA/adapter、合成数据、模块化组件（检索器/验证器/监控器）做“**小改动大收益**”。（这一块更多是“方法论支撑”，不单独押注 best-paper 叙事）
* 痛点：容易被认为工程；需要与 **机制解释/理论保证** 绑定才够强。

### 1.7 多智能体/工具使用（agents / tool use / planning / search）

* 主线：现实环境 benchmark（WebArena/GAIA）、工具调用稳定评测（StableToolBench）、以及软件工程 agent（SWE-agent）。([ICLR 会议录][13])
* 痛点：评测不稳定、环境漂移、奖励信号噪声大；“看似进步”但可复现性不足。([arXiv][14])
* 重要性：若能把 agent 的“可靠性/安全性”形式化（尤其与 verifier / conformal 结合），很有 best-paper 潜力。([arXiv][15])

### 1.8 检索与记忆（RAG / long-context / memory systems）

* 主线：RAG 从“拼接文档”走向 **纠错型检索（CRAG）、自反思检索（Self-RAG）、层级摘要树（RAPTOR）、GraphRAG**。([arXiv][16])
* 痛点：检索错了会“越检索越幻觉”；鲁棒性与评测仍混乱。([arXiv][16])
* 重要性：这里“新评测维度/新 failure mode”空间大，适合做 best-paper 级 evaluation + principle。([arXiv][17])

### 1.9 结构化生成/程序归纳（code / formal methods / theorem proving）

* 主线：用 Lean/形式化验证做 **机器可检验的正确性**：AlphaProof（IMO 级形式化推理）、verified code generation benchmark（CLEVER）、Lean4 证明评测（VeriBench-FTP）。([Nature][18])
* 痛点：交互式证明搜索难、数据稀缺；但**可验证性极强**。([NeurIPS][19])
* 重要性：NeurIPS/ICLR 对“可验证推理”非常买账；容易做出“机制+严格评测”。([NeurIPS][20])

### 1.10 安全与红队（jailbreak / adversarial robustness）

* 主线：标准化红队评测（HarmBench、JailbreakBench）、agent 场景的恶意任务（AgentHarm）、以及新的安全基准（如 AILuminate）。([arXiv][21])
* 痛点：攻防共演、评测可迁移性差；“防住某类 jailbreak”常伴随可用性下降。([arXiv][21])
* 重要性：若能把安全与 agent / tool-use / verifier 统一成 **可形式化的风险控制**，很可能冲 best paper。([arXiv][22])

## 3. 深入对比（每方向：现状 → 瓶颈 → 争议 → 可突破点｜含 6–12 篇代表作）

### 3.0 Step C：找 gap 的“系统化工具箱”（我用哪些策略找到了哪些缺口）

下面每个方向的“可突破点”都会显式标注使用了哪几种 gap 策略：

1. **Failure modes map**：画出失败模式（如 reward hacking、域迁移失效、检索错导致幻觉）。
2. **Assumption stress test**：把隐含假设写出来（如 exchangeability、verifier 不被操纵、benchmark 无污染）。
3. **Counterfactual baseline**：构造更强/更坏的反事实（如“攻击者专门优化 verifier 分数但答案错”）。
4. **Unifying principle**：把分散方法统一到一个原则（如“验证=风险控制/置信下界驱动计算分配”）。
5. **New eval lens**：引入新指标/新基准让旧方法暴露问题（如“单调性、coverage、exploitation rate”）。
6. **Scaling/complexity lens**：看推理计算/采样数增长时是否单调、复杂度是否可控。

---

### 3.1 D1 生成式 Verifier & PRM（推理/agent 的 test-time scaling）

**代表作（6–10）**

* **Generative Verifiers / GenRM**：把 reward modeling 改写为 next-token prediction 的“生成式验证”。([arXiv][1])
* **Agent Process Reward Models（AgentPRM）**：面向 agent 的过程奖励建模与 rollouts。([arXiv][15])
* **R-PRM（EMNLP 2025）**：指出现有 PRM 直接输出分数的限制，并改进学习/评估效率。([ACL Anthology][27])
* **Min-Form Credit Assignment…（OpenReview）**：分析 PRM 诱导的 reward hacking，并从 credit assignment 角度修正。([OpenReview][2])
* **VersaPRM（ICML 2025 页面）**：强调 PRM 的跨域泛化问题。([ICML][28])
* **Self-Rewarding Language Models**：LLM-as-a-judge 自给自足的奖励信号。([arXiv][29])

**现状**

* “推理增强”从 CoT 提示转向 **生成→验证→选择/搜索** 的可控流水线。([arXiv][1])
* PRM 逐步从“数学推理”扩展到 agent 交互。([arXiv][15])

**瓶颈**

* （Failure modes map）**exploitation/reward hacking**：策略学会“讨好 verifier”。([OpenReview][2])
* （Assumption stress test）默认“verifier 分数≈正确率/成功率”但经常不成立（尤其跨域）。([ICML][28])
* 数据：步骤级标注昂贵，rollout 估计方差大。([arXiv][15])

**争议/负结果**

* GenRM/PRM 可能 **提升 test-time** 但 **破坏 fine-tuning 稳定性**（因为被 exploit）。([OpenReview][2])
* “LLM-as-a-judge”在不同模型/不同任务下可靠性波动。([arXiv][29])

**可突破点（gap）**

* （Counterfactual baseline + New eval lens）建立 **“PRM-Hack”反事实评测**：攻击者专门优化 PRM 分数但保持最终答案错误，量化 exploitation rate。
* （Unifying principle）把 PRM 输出从“分数”升级为 **可校准成功概率的置信下界**，用于 **风险控制式** test-time scaling。([arXiv][5])
* （Scaling lens）研究 **采样数 N 增长时准确率是否单调**；若不单调，说明 verifier 被 exploit（这是可证伪预测）。

---

### 3.2 D2 Reward Model “好老师”理论（超越 accuracy）

**代表作（6–9）**

* **What Makes a Reward Model a Good Teacher?**：reward variance 决定 RLHF 优化景观，且“准确也可能很难优化”。([OpenReview][23])
* **Generalizable Principle-Following Reward Models**：reward model 的“原则泛化”。([OpenReview][30])
* **Self-Rewarding Language Models**：迭代 DPO + 自评判奖励。([arXiv][29])
* **LLM Critics Help Catch LLM Bugs / Critic 系列**：用 critic 改善人类/模型协作质量。([arXiv][31])
* **Uncertainty Calibration of PRMs（NeurIPS 2025 poster）**：把 PRM 输出校准为成功概率并做 instance-adaptive scaling。([NeurIPS][32])
* （可选补充）从 preference/teacher 角度引入 ORPO/SimPO 的 implicit reward 讨论。([arXiv][33])

**现状**

* 研究开始把 reward model 当作“**教学信号**”而非静态分类器：要预测“能不能带动策略学”。([OpenReview][23])

**瓶颈**

* （Assumption stress test）传统只用 accuracy/AUC 评 RM，隐含假设“准确→好教”，已被理论否定。([OpenReview][23])
* RLHF 端到端评估成本高，导致“RM 指标”难统一。

**争议/负结果**

* 更复杂 RM 不一定更好；跨 policy/跨模型迁移差。([OpenReview][23])

**可突破点（gap）**

* （Unifying principle）提出 **Teacher Quality** 的统一度量：同时刻画 *rank accuracy* 与 *reward variance / gradient signal*，并能预测下游 RL 收敛速度。([OpenReview][23])
* （New eval lens）把评测从“静态准确率”改为“**在固定计算预算下，策略提升曲线**”（learning curves）+ 显著性检验。

### 3.3 D3 离线偏好优化新范式（ORPO/SimPO/KTO/SimPER）

**代表作（6–10）**

* ORPO（reference-free monolithic preference optimization）。([arXiv][8])
* SimPO（reference-free implicit reward + margin）。([arXiv][33])
* KTO（前景理论/utility，强调 inductive bias）。([arXiv][9])
* SimPER（ICLR 2025，超参自由）。([OpenReview][34])
* Weighted-Reward Preference Optimization（ICLR 2025 论文页）。([ICLR 会议录][35])
* CPO（对比偏好优化，尤其 MT）。([arXiv][36])
* 额外：BNF 等偏好对齐变体（ICLR 2025）。([ICLR 会议录][37])

**现状**

* 明确趋势：**“不做 RL 也能对齐”**，并把优化问题写成更稳定的监督式目标。([arXiv][8])

**瓶颈**

* （Failure modes map）指标涨但 **长度控制、偏好偏置、领域外泛化** 不稳。([arXiv][9])
* （Assumption stress test）默认偏好数据代表真实效用；但数据分布变化就失效。

**争议**

* 是否“又一个 loss”：缺少统一解释时难冲 best paper。([arXiv][9])

**可突破点（gap）**

* （Unifying principle）把 ORPO/SimPO/KTO 统一为 **“隐式 reward 形状 + utility 偏置”** 的 family，给出可检验预测：何时哪个 bias 更优。([arXiv][9])
* （Counterfactual baseline）构造“反事实偏好数据”（系统性偏置）评估鲁棒性，而不是只看 AlpacaEval/MT-Bench。

### 3.4 D4 不确定性与 Conformal（含域迁移/选择性预测）

**代表作（6–10）**

* SConU（Selective Conformal Uncertainty）。([arXiv][5])
* DS-CP（Domain-Shift-Aware Conformal Prediction）。([arXiv][38])
* NeurIPS 2025：Revisiting Uncertainty Estimation and Calibration…（系统性评测）。([NeurIPS][4])
* SelectLLM（选择性预测/校准）。([NeurIPS][3])
* PRM 的不确定性校准与 instance-adaptive scaling。([NeurIPS][32])
* Self-consistency 用于 calibration（EMNLP 2024 findings）。([ACL Anthology][39])
* HalluLens（幻觉评测）。([ACL Anthology][40])

**现状**

* 校准从“温度缩放”走向 **风险控制/coverage 约束**；并开始系统评估“不同模型/量化/推理能力对不确定性”的影响。([NeurIPS][4])

**瓶颈**

* （Assumption stress test）conformal 依赖交换性/同分布；域迁移下容易无效，DS-CP 尝试修补。([arXiv][38])
* （Failure modes map）推理链条依赖导致“逐步置信”组合困难（多步 error control）。

**争议**

* coverage 保证是否真的对应“真实风险”（尤其开放域问答）。

**可突破点（gap）**

* （Unifying principle + New eval lens）把 **“不确定性→计算分配/拒答/检索触发”** 统一到一个决策框架，并用 **selective risk–coverage 曲线** 评测。([NeurIPS][3])
* （Scaling lens）研究“采样数↑时，coverage 是否保持、风险是否下降”，建立可证伪预测。

### 3.5 D5 SAE 机制解释（特征发现/steering/泛化）

**代表作（6–10）**

* Scaling & evaluating SAEs（研究 SAE scaling/死特征等）。([OpenReview][6])
* Sparse Autoencoders Find Highly Interpretable Features in LLMs。([OpenReview][7])
* Claude 3 Sonnet 特征提取（transformer-circuits）。([变压器电路][41])
* At the Edge of Understanding: SAEs Trace OOD behavior（OpenReview）。([OpenReview][42])
* Encourage or Inhibit Monosemanticity?（讨论单语义性与训练权衡）。([arXiv][43])
* SAEs 用于发现未知概念（2025）。([arXiv][44])

**现状**

* SAE 成为“无监督特征字典”，可用于解释与控制。([OpenReview][7])

**瓶颈**

* （Assumption stress test）“特征=概念”的假设并不总成立；同一特征跨层/跨模型稳定性不足。([OpenReview][6])
* 工程：训练巨大 SAE、保存激活、dead latents 处理。([OpenReview][6])

**争议**

## 4. 候选课题（3–5 个“最可能冲 best paper”的题）

### 4.1 课题 A（主推候选）：**对抗鲁棒 + Conformal 的 PRM：把 verifier 变成“可校准的风险控制器”**

* 核心问题：**PRM/Verifier 在 test-time scaling 与 RL fine-tuning 中经常被 exploit，导致非单调甚至退化；如何让它在域迁移与对抗样本下仍可用，并给出可验证保证？**
* 关键缺口（gap）：现有 PRM 研究要么关注更强分数，要么做后处理校准，但缺少 **“对抗可用性 + 统计保证 + 单调性评测”** 的统一闭环。([OpenReview][2])
* Best-paper 支点：**Unifying principle + New eval lens + 部分理论**（风险控制/coverage/单调性），并构建反事实攻击基准。
* Timing：PRM/Verifier 爆发期且已出现 reward hacking 专门论文与 PRM 校准论文，正是“第二阶段：补机制与可信度”的窗口。([OpenReview][2])

* 支点：强理论 + 可证伪预测（学习曲线/梯度景观），并给出可复现算法。([OpenReview][23])
* Timing：NeurIPS 2025 spotlight 已把问题“钉死”，后续最缺的是 **把理论变成可用方法**。([OpenReview][23])

### 4.3 课题 C：**“污染校正后的能力评测”统一框架：动态无污染 + 因果量化**

* 核心问题：**如何给 LLM 能力评测一个“可比较、可审计”的分数（含污染风险与置信区间）？**

* 核心问题：**agent 在多步工具执行中如何保证“只做被允许的事”，同时不牺牲能力？**

> * 可验证/可复现（机器可检验 correctness、coverage、单调性）

### 5.0 在选择主推 idea 前：必须回答的关键问题（可见）

**Q1：最接近的 3 篇/3 类工作是谁？我们多了什么“不可替代”的东西？**

1. **GenRM / Generative Verifiers**：把 verifier 生成化、提升验证能力。([arXiv][1])

   * 我们的不可替代：**把 verifier 从“分数/排序器”升级为“可校准风险控制器”**（conformal 下界 + 域迁移处理），并显式对抗“verifier exploit”。
2. **Min-Form Credit Assignment（PRM reward hacking）**：解释 PRM 被 hack 的原因并提出修正。([OpenReview][2])

   * 我们的不可替代：在 credit assignment 之外，引入 **对抗反事实评测 + 对抗数据闭环训练**（攻击者生成“高分错解”），并用统计保证把“可用性”量化。
3. **PRM 不确定性校准与 instance-adaptive scaling（NeurIPS 2025 poster）**：对 PRM 输出做校准并动态分配计算。([NeurIPS][32])

   * 我们的不可替代：把校准从回归式后处理扩展为 **conformal（含域迁移/选择性）** 的保证，并引入 **PRM-Hack 基准**，验证在对抗分布下仍可靠。([arXiv][38])

**Q2：贡献属于新机制/新理论/新评测/新问题定义哪一种？为什么足够强？**

* 组合拳：

  * **新问题定义**：把 PRM 视为“风险控制器”，目标不是分数高，而是 **在预算增长下保持单调改进，并对错误风险给出可校准保证**。
  * **新评测（核心）**：PRM-HackBench + 单调性曲线 + coverage/ selective risk。
  * **新机制**：对抗闭环训练让 PRM “抗 exploit”；conformal 让 PRM “可审计”。
  * **部分理论**：coverage / 风险控制（在合理假设下），并给出违反假设时的可观测失效信号（可证伪）。([arXiv][5])

**Q3：在算力约束下最小闭环是什么（2–3 周出继续/停止信号）？**

* 2–3 周 MVP：

  1. 复现一个公开 PRM/verifier（或直接用 GenRM）在 **数学推理** 的 Best-of-N 提升。([arXiv][1])
  2. 做一个简单攻击者（提示/搜索/小 RL）生成“高 PRM 分但错”的轨迹，证明 **reward hacking 可复现**。([OpenReview][2])
  3. 加入 **对抗训练 + conformal 下界** 后，看 **(i) 攻击成功率下降、(ii) accuracy 随 N 更接近单调**。
* 只要 (2) 与 (3) 的现象成立，就值得全力推进（很强叙事）。

**Q4：best paper 级叙事一句话能讲清吗？**

* “我们把 PRM 从脆弱的启发式打分器，变成 **对抗鲁棒且可校准的风险控制器**；从而在推理 test-time scaling 中实现 **可审计的单调改进**。”

### 5.1 一句话主张（thesis claim）

**Conformalized Adversarial PRM（CAP-PRM）** 通过“对抗闭环训练 + 域迁移感知的 conformal 置信下界”，让过程奖励模型在 reward hacking 与分布偏移下仍可用，并使 test-time scaling 的性能随计算预算呈可审计的单调改进。

---

### 5.2 创新点列表（novelty checklist｜≥5 条，含与最接近 3 类工作逐项对比）

1. **把 PRM 目标从“更准的分数”改为“风险控制器”**：输出 **成功概率的置信下界**（可用于拒答/继续采样/回滚）。对比：传统 PRM/GenRM 主要做排序/打分。([arXiv][1])
2. **PRM-HackBench（新评测维度）**：系统化构造“高分错解”的对抗轨迹，度量 exploitation rate、单调性破坏点、以及防御泛化。对比：现有工作多在自然分布评测，较少用强反事实基线。([OpenReview][2])
3. **对抗闭环训练（attacker–verifier game）**：攻击者专门优化 PRM 分数但约束最终错误，用于生成 hard negatives；对比：仅靠 credit assignment 修正不足以覆盖“提示诱导/分布偏移下的 exploit”。([OpenReview][2])
4. **域迁移感知 conformal**：把 DS-CP / 选择性 conformal 的思想用于 PRM 的概率下界（而不仅是最终答案），并给出失效诊断信号。([arXiv][38])
5. **单调性作为核心指标（scaling lens）**：关注 accuracy vs compute(N) 的形状（单调/非单调/拐点），把“越算越错”变成可复现、可解释的 failure mode。
6. **跨域验证**：用 VersaPRM 指出的跨域问题作为主战场之一，评测“训练于数学→测试于代码/工具”时 CAP-PRM 的鲁棒性。([ICML][28])

---

### 5.3 核心洞察（为什么不是工程堆砌，而是新视角/机制）

* 旧视角：PRM 是“更聪明的评分器”。
* 新视角：PRM 是 **决策系统的风险仪表盘**：

  * 它必须对 **对抗操纵** 鲁棒（否则越用越糟）。([OpenReview][2])
  * 它必须能给出 **可校准的置信界**（否则无法做可靠的停止/拒答/计算分配）。([arXiv][5])
* 因此贡献不是“多训一点/多采样一点”，而是把 PRM 变成 **可审计、可证伪、可用于风险控制** 的组件。

### 5.4 方法清晰定义（伪代码级别）

#### 模块

* **Reasoner**：基础 LLM（开源 7B–14B 级别）生成推理轨迹。
* **PRM / Verifier**：对“部分轨迹”输出成功概率 (\hat{p}_\theta)。可用 GenRM 式生成式验证。([arXiv][1])
* **Attacker**：生成“高 PRM 分但错误”的轨迹（反事实基线/对抗数据生成器）。
* **Conformal Calibrator**：把 (\hat{p}*\theta) 转成 **下界** (LB*\alpha)，并在域迁移下重加权（DS-CP）。([arXiv][38])

#### 训练数据：用 rollout 构造步骤级目标（避免人工 step-label）

对每个问题 (x)，采样轨迹 (y=(s_1,\dots,s_T))。
对每个前缀 (y_{\le t})，做 (K) 次从该前缀继续生成的 rollout，得到成功率：
[
\hat{q}(x,t)=\frac{1}{K}\sum_{k=1}^K \mathbf{1}[\text{final_answer correct}]
]
这就是 PRM 的监督信号（“从这一步继续走，成功概率多大”），与 AgentPRM 的思路一致。([arXiv][15])

#### 对抗样本：PRM-Hack 轨迹

Attacker 目标：最大化 PRM 分数但最终错误：
[
\max_{y} \sum_{t} \hat{p}*\theta(x, y*{\le t}) \quad \text{s.t.}\quad \text{final incorrect}
]
（实现上可用：prompt 攻击、采样搜索、或轻量 RL/EM 式迭代；不需要白盒梯度。）

#### 伪代码（CAP-PRM 训练）

```
Input: tasks D = {x}, reasoner π, verifier fθ, attacker aφ
for epoch = 1..E:
  # 1) 采样自然轨迹并 rollouts 得到步骤成功率
  for x in minibatch(D):
    y ~ π(x)                      # 推理轨迹
    for t in 1..T:
      q_hat[x,t] = RolloutSuccess(π, x, prefix=y<=t, K)
      add (x, y<=t, q_hat[x,t]) to S_nat

  # 2) 生成对抗轨迹（高分错解）
  y_adv ~ aφ(x, fθ)  subject to final incorrect
  build step targets q_hat_adv via rollouts or set q_hat_adv=0
  add (x, y_adv<=t, q_hat_adv) to S_adv

  # 3) 训练 verifier：回归成功率 + 对抗分离 margin
  minimizeθ  L_reg(fθ(x, prefix), q_hat) + λ * L_margin(S_adv vs S_nat)

# 4) Conformal calibration（含域迁移加权）
Fit conformal set to output LBα(x, prefix) using DS-CP / SConU-style tests
Return: fθ + calibrator
```

### 5.5 训练/推理细节（超参、损失、复杂度）

**Verifier/PRM 形态**

* 推荐：**生成式 verifier（GenRM 风格）**：让模型输出“是否能最终成功/是否正确”的 token 概率，再映射到 (\hat{p})。([arXiv][1])

**损失**

* 回归损失：(L_{reg}=\text{Huber}(\hat{p}_\theta - \hat{q})) 或 BCE（把 (\hat{q}) 当软标签）。
* 对抗分离：

  * margin ranking：让自然轨迹前缀的 (\hat{p}) 高于对抗轨迹前缀；
  * 或 focal loss 强化 hard negatives。
* 可选：引入“min-form credit assignment”式的分配，以减少 hacking（作为 ablation）。([OpenReview][2])

**Conformal 校准**

* 非一致性分数：(s=1-\hat{p}_\theta)。
* 置信下界：(LB_\alpha = 1 - Q_{1-\alpha}(s))（分位数来自校准集）。
* 域迁移：对校准样本按与测试 prompt 的相似度重加权（DS-CP 思路）。([arXiv][38])
* 若检测到“样本偏离校准分布”，触发选择性机制（SConU 思路）：给出“无法保证”的信号或更保守下界。([arXiv][5])

**推理（test-time scaling）**

* 在生成过程中，按步计算 (LB_\alpha(x, y_{\le t}))：

  * 若 (LB_\alpha) 高于阈值 → **停止继续采样/停止扩展**（或直接提交）。
  * 若 (LB_\alpha) 低且不确定 → 增加采样、回溯、或触发工具/检索。
* 复杂度：主要开销来自 rollouts/采样。CAP-PRM 的目标是**用更可靠的 LB 减少无效采样**。

### 5.6 至少 3 个强基线（含最难打败的 baseline）

1. **GenRM / Generative Verifier + Best-of-N**（最硬 baseline）。([arXiv][1])
2. **PRM + 后处理校准 + instance-adaptive scaling**（NeurIPS 2025 poster 思路）。([NeurIPS][32])
3. **Min-form credit assignment 修正的 PRM fine-tuning**（解决 hacking 的代表线）。([OpenReview][2])
   （可选 4）**LLM-as-a-judge/self-rewarding** 作为弱监督/评估基线。([arXiv][29])

### 5.7 Minimum Viable Experiments（≥6｜每个都有可证伪预期）

> 先在“可自动验真”的任务上闭环：数学（GSM8K/MATH）、代码（HumanEval/MBPP/SWE-bench 子集）、工具环境（StableToolBench/WebArena 子集）。

1. **复现：GenRM/PRM 的 Best-of-N 曲线**

* 指标：accuracy vs N（采样数）、token 成本。
* 预期：能复现显著提升（否则先别谈新方法）。([arXiv][1])

2. **PRM-HackBench：构造对抗“高分错解”并量化 exploitability（新评测）**

* 指标：attack success rate、exploitation gap（对抗轨迹分数 - 自然错解分数）。
* 可证伪预期：**现有 PRM 在某些任务/阈值下可被稳定 hack**。([OpenReview][2])

3. **对抗训练是否提升单调性（关键现象）**

* 指标：accuracy vs N 的**单调性破坏次数**、拐点位置。
* 预期：CAP-PRM 的曲线更接近单调；基线出现“越算越错”的区域（可证伪）。

4. **Conformal 下界的 coverage 与 selective risk 曲线**

* 指标：在不同 (\alpha) 下的 miscoverage、风险-覆盖曲线。
* 预期：自然分布上 coverage 接近目标；域迁移下 DS-CP/SConU 版优于 vanilla。([arXiv][38])

5. **跨域泛化：数学→代码/工具（针对 VersaPRM 指出的痛点）**

* 指标：跨域 accuracy、calibration drop、hackability drop。
* 预期：对抗训练 + LB 能显著降低跨域 hackability，即使 accuracy 提升有限也能构成贡献。([ICML][28])

6. **Ablation（必须细）**

* 去掉 attacker（无对抗数据）
* 去掉 conformal（只用 raw score）
* 去掉 DS-CP（不做域迁移）
* 去掉 rollout soft target（只用最终 0/1）
* 预期：每个模块对应不同 failure mode（可证伪）。([arXiv][15])

7. **机制验证：攻击轨迹的“错误类型地图”**（Failure modes map 的量化版）

* 指标：错误类型（逻辑跳步/符号错/伪造中间结论等）与 PRM 分数的关系。
* 预期：CAP-PRM 对“伪装正确的中间步骤”更敏感（若不成立，说明对抗样本没覆盖）。

8. **计算效率实验：同等 token 预算下谁更强**

* 预期：CAP-PRM 在相同预算下更优，或在相同准确率下更省 token（两者满足其一即可）。

---

### 5.8 预期会失败的设置（≥2｜失败也有价值）

1. **无法自动验真的开放式任务**（写作/主观偏好）：rollout 成功率 target 不可靠 → conformal 也难定义“正确”。

* 价值：明确方法适用边界，推动“可验证任务优先”的评测主张。

2. **极端域迁移且校准集完全不覆盖**：DS-CP/SConU 仍可能过度保守或失效。([arXiv][38])

* 价值：可以把“失效检测”作为贡献：何时系统应输出“无法保证”。

---

### 5.9 潜在风险与备选路线（Plan B / C）

* 风险 R1：对抗样本生成成本高/质量差

  * **Plan B**：先用 prompt 搜索/温度扫描/启发式构造最小攻击集；再逐步上 attacker。
* 风险 R2：conformal 下界太保守，导致性能下降

  * **Plan B**：做“选择性触发”：只在检测到 OOD/高风险时启用更保守下界。([arXiv][5])
* 风险 R3：单调性提升不明显

  * **Plan C**：把贡献聚焦为 **“PRM-HackBench + exploitation-aware metric”**（新评测/新 failure mode），并把 CAP-PRM 作为 strong baseline。

---

### 5.10 诊断树（debug roadmap｜如果结果不如预期怎么迭代）

**现象 A：攻击者几乎造不出“高分错解”**

* 可能原因：PRM 已很强 / 攻击空间太小 / 约束太严
* 行动：

  * 放宽攻击：允许“部分正确但最终错”、允许更长轨迹；
  * 换任务：从数学转到更易 exploit 的代码/工具流程；
  * 先验证 baseline hacking 是否可复现（对照 Min-Form paper 的描述）。([OpenReview][2])

**现象 B：对抗训练后 accuracy 下降**

* 可能原因：hard negatives 过强导致过拟合“拒绝/保守”
* 行动：

  * 调整对抗比例（adv:nat）、使用 curriculum（先弱后强）
  * 让对抗样本只影响 margin，不影响回归 target

**现象 C：coverage 不达标（conformal 失效）**

* 可能原因：交换性破坏 / 校准集偏移
* 行动：

  * 引入 DS-CP 重加权；
  * 加 SConU 的 outlier 检测，输出“无法保证”而不是假装有保证。([arXiv][38])

**现象 D：单调性仍差**

* 可能原因：攻击者没覆盖真实 exploit；或 PRM 学到 spurious cues
* 行动：

  * 把 exploit 轨迹做聚类，补齐缺失模式（failure map）
  * 引入“反事实基线”：同答案但打乱推理步骤，检查 PRM 是否被表面格式骗过

## 6. 执行计划（里程碑 + 资源估算 + 实验清单）

### 6.1 数据与基准选择（含理由）

**必须优先：可自动验真（falsifiable）**

* 数学/推理：GSM8K、MATH（或同类公开集）；用于验证 PRM/单调性。
* 代码：HumanEval/MBPP + **SWE-bench 子集**（真实 repo 修复，最贴近 agent）。([ICLR 会议录][10])
* 工具/网页：StableToolBench（稳定评测）、WebArena（真实网页）、GAIA（通用助手）。([arXiv][14])

**对抗/安全（可选扩展但很加分）**

* AgentHarm：把 CAP-PRM 的“风险控制”用于恶意任务拒绝与过程约束。([arXiv][22])

---

### 6.2 模型/方法细节（算法步骤、训练细节、推理流程）

**模型建议（不绑定具体牌子，按你们生态选）**

* Reasoner：开源 7B–14B 指令模型（便于复现）。
* PRM/Verifier：

  * 方案 1：同尺寸或更小（1.5B–7B）模型做 GenRM 风格 verifier；
  * 方案 2：更小 encoder（若只做分类）+ 解释性较弱但更快。([arXiv][1])

**训练关键超参（起步值）**

* LoRA：rank 16/32；target modules = attention + MLP；bf16。
* rollout：每前缀 (K=8)（先小后大做消融）。
* 对抗比例：adv:nat = 1:3 起步；做 curriculum。
* 校准集：从训练集划出 10%（并保留一个“域外校准集”用于 DS-CP）。([arXiv][38])

---

### 6.3 评测协议（主指标、鲁棒性、统计显著性、机制验证）

**主指标**

* accuracy / pass@k（代码/测试）
* 预算曲线：accuracy vs N、accuracy vs tokens
* 单调性：曲线非单调次数、拐点 N*
* calibration：ECE、selective risk–coverage、conformal miscoverage

**鲁棒性**

* 对抗：PRM-HackBench attack success rate
* 域迁移：train on math → test on code/tool；以及 prompt 风格变化

**显著性**

* bootstrap 置信区间（任务级 resample）
* 多随机种子（至少 3）
* 报告 effect size，而不是只报均值

**机制验证**

* failure map：把攻击样本按错误类型聚类，并量化 CAP-PRM 的分数抑制效果（对应 5.7-7）。

---

### 6.4 资源估算（显存、训练时间、GPU 天数上限、并行策略）

你们总资源：**16×80GB A800**。目标是不超过“几十 GPU-days”。

**建议的并行策略**

* 用 8 卡跑 PRM LoRA 训练（FSDP/ZeRO）
* 另 8 卡跑推理采样（vLLM/TensorRT-LLM）与攻击样本生成
* 校准与评测可在推理卡上穿插执行

**粗量级预算（保守上限）**

* PRM 训练：10 次实验 × 每次 0.5 GPU-day ≈ **5 GPU-days**
* rollout/对抗数据生成：≈ **10–20 GPU-days**（取决于 K 与任务规模）
* 全量评测 + ablation：≈ **10–15 GPU-days**
* 合计：**约 25–40 GPU-days**（在你们资源下很舒适）

---

### 6.5 复现与工程组织（实验配置、日志、超参搜索）

* 配置：Hydra/YAML + 固定 seed；每次实验输出 git commit hash + 数据版本号
* 数据版本：把 (x, trajectory prefix, rollout success, attacker flag) 保存成可重放的 parquet
* 评测：统一 harness（同一采样器、同一温度/最大 token）
* 超参：优先做 **小网格**（adv 比例、K、λ、α）+ 早停
* 日志：记录每一步 prefix 的 (\hat{p})、(LB_\alpha)、最终正确性 → 支持机制分析

### 7.2 摘要骨架（按 best-paper 写法）

* 背景：PRM/verifier 成为推理与 agent 的核心组件，但存在 reward hacking、域迁移失效与非单调 scaling。([OpenReview][2])
* 问题：现有方法缺少把“鲁棒性 + 统计保证 + 计算分配”统一起来的框架。([NeurIPS][32])
* 方法：CAP-PRM（对抗闭环训练 + DS-aware conformal 下界）
* 新评测：PRM-HackBench + 单调性曲线 + selective risk/coverage
* 结果：在数学/代码/工具任务上降低 hackability、提升或保持准确率，并在预算增长下更单调；conformal 指标满足目标 coverage（或在失效时可靠报警）。([arXiv][5])
* 影响：把 verifier 从启发式评分器变成可审计风险控制器，为可靠推理与 agent 提供通用组件。

---

### 7.3 贡献点 bullets（建议写在 Introduction 的末尾）

* 提出 **PRM 作为风险控制器** 的问题定义，并给出可审计目标（单调性 + coverage）。
* 提出 **PRM-HackBench**：系统化对抗轨迹评测，量化 exploitation。
* 提出 **CAP-PRM**：对抗闭环训练 + DS-aware conformal 下界。
* 给出跨域与机制分析：哪些错误模式被修复、哪些仍失败。
* 开源：数据、评测脚本、攻击生成器、训练配置（确保可复现）。
