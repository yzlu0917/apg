## 1. 方向地图（含主题覆盖）

### 1.1 我覆盖的 8 个主题，以及为什么这样划分

我覆盖了 8 个主题；其中把“高效学习”并入“训练与优化新范式”，因为过去 12 个月里很多真正有影响力的高效学习结果已经体现在 small-data reasoning、reference-free preference learning、credit assignment 和 adaptive compute 里；把“安全与红队”并入“多智能体/工具使用”，因为近期安全前沿最活跃的就是 web/computer-use agent 的 prompt injection 与 environment attack。([arXiv][1])

### 1.2 Step A：广撒网后的主题主线图

#### T1. 推理与验证（reasoning / verifier / self-correction / test-time scaling）

* 主线：从 outcome-only RLVR/R1，到 test-time scaling、verifier-guided search、generative verifier、adaptive verification，目标是把“多想一会儿”变成系统化的 compute allocation。DeepSeek-R1、s1、SETS、Heimdall、solve-vs-verify tradeoff、imperfect verifier 下的 scaling law 是这条线的核心。([arXiv][2])
* 痛点：正确答案不等于正确过程；verifier 不完美时，重采样和自验证很快遇到上限；“为什么有效”仍不清楚。([arXiv][3])
* 重要性：这是 2025–2026 最接近“能力前沿”的主线，但复制者极多，单纯改 loss 很难形成 best-paper 级 moat。([arXiv][1])

#### T2. 可靠性与对齐（robustness / honesty / calibration / uncertainty / abstention）

* 主线：从“能不能答对”转到“知不知道自己不知道”，包括 abstention、uncertainty calibration、honesty、alignment faking、reasoning 与 refusal/abstention 的张力。AbstentionBench 显示 reasoning FT 会显著恶化 abstention；多个不确定性 benchmark 开始把 calibration 从附属指标变成主任务。([开放评论][4])
* 痛点：现有方法常把“有链路地自信地错”当成进步；很多 calibration 结果对 domain、prompt style、judge choice 非常敏感。([arXiv][5])
* 重要性：这条线社区兴趣越来越强，而且适合做“新评测 + 机制解释”型论文。([开放评论][4])

#### T3. 表征与机制解释（mechanistic interpretability / representation learning）

* 主线：SAE 已从“局部可解释 feature”发展到“跨模型通用 feature space”“自动解释百万 feature”“电路 tracing”“生物学式系统图谱”。Anthropic 的 Claude 3 Sonnet monosemanticity、Biology of an LLM、Tracing the Thoughts，以及 universal feature space、Matryoshka SAE 是代表。([Semantic Scholar][6])
* 痛点：解释很多，但“可操作的机制结论”仍少；SAE feature 是否 faithful、是否足够 task-grounded，依然有理论和方法争议。([arXiv][7])
* 重要性：这是最像 best-paper 的“机制支点”，但需要和一个能力/训练问题绑在一起，否则容易变成漂亮但孤立的分析。([变压器电路][8])

#### T4. 训练与优化新范式（RLHF/RLAIF alternatives, off-policy RL, direct preference, credit assignment, efficient learning）

* 主线：DPO/ORPO/SimPO 之后，社区转向 RLVR、GRPO/2-GRPO、R1-Zero 式训练、PRIME/PURE/CAPO 这样的细粒度 credit assignment，以及“小数据也能引出推理”的 LIMO/s1。很多新结果说明：真正稀缺的是 credit，而不是更多 token。([arXiv][9])
* 痛点：length bias、implicit DPO equivalence、reward hacking、credit diffusion 很普遍；dense reward 方法往往靠昂贵 PRM 或 LLM judge，faithfulness 存疑。([arXiv][10])
* 重要性：最贴近顶会主赛道，而且在你们的算力下能做扎实对比，但必须避免“又一个 RL loss”。([arXiv][1])

#### T5. 数据合成与评测（synthetic data, evaluation, contamination, benchmark design）

* 主线：从 static benchmark 转向 live、contamination-aware、counterfactual、diagnostic benchmark。LiveBench、LiveCodeBench、MMLU-Pro、HLE、HLE-Verified、MMLU-CF、RAGChecker 都体现出“旧 benchmark 已经不够用了”。([开放评论][11])
* 痛点：很多 SOTA 排名受污染、题目噪声、评测代理不稳定影响；benchmark 一旦静态化，很快失真。([arXiv][12])
* 重要性：这是形成论文 moat 的最佳配件；“新原则 + 新评测”比“新 loss + 旧 benchmark”更像 best paper。([开放评论][11])

#### T6. 多智能体 / 工具使用（agents, tool use, planning, search）并纳入 agent safety

* 主线：从 SWE-agent、OSWorld，到 web research agent（Search-R1、WebSailor、ReSum、Deep Research Bench），再到 agent prompt injection / WASP / OS-Harm。焦点从“能否调用工具”转到“长程任务中如何规划、检索、总结、抗攻击”。([arXiv][13])
* 痛点：环境噪声大、评测成本高、复现难；安全攻击与能力提升强耦合。([arXiv][14])
* 重要性：顶会热，但做 best paper 需要很强基准和工程。对你们来说适合做评测/机制切口，不适合纯 agent stack。([arXiv][15])

#### T7. 检索与记忆（RAG, long-context, memory systems）

* 主线：LongBench v2、RULER、MemoRAG、Self-Route、MInference、SeaKR 表明“长上下文不自动等于好推理”，现在转向 retrieval-aware routing、memory-aware uncertainty、长程 grounding。([arXiv][16])
* 痛点：长上下文 reward 很稀疏；模型经常“看到了但没用到”；LC vs RAG 的选择仍缺统一原则。2026 已经有工作直接指出 long-context RL 需要 verifiable grounding。([arXiv][17])
* 重要性：如果能把 reasoning credit assignment 和 grounding reward 联起来，潜力很大。([arXiv][18])

#### T8. 结构化生成 / 程序归纳（code, formal methods, theorem proving）

* 主线：formal theorem proving、verified code reasoning、hierarchical decomposition、self-play theorem provers、DeepSeek-Prover-V2、Goedel/Leanabell/StepFun/PhysProver。优势是 outcome verifiable，天然适合做 credit assignment。([arXiv][19])
* 痛点：生态碎片化、数据清洗和 proof assistant 工程门槛高；如果只追 leaderboard，很容易沦为 domain-specific。([arXiv][20])
* 重要性：这是最适合做“强验证 + 强机制”的试验田，也最容易得到可证伪结论。([arXiv][19])

### 3.1 D1 推理后训练 + verifier-guided TTS

**现状**

* Outcome-only RLVR/R1 把“会推理”重新定义成“在可验证任务上用结果奖励 post-train”；test-time scaling 与 verifier-guided search 成为配套。([arXiv][2])
* 自验证、adaptive verification、compute allocation 都在解决“推理 token 怎么花得值”。([arXiv][27])

**瓶颈**

* outcome reward 太稀；长答案/冗余轨迹能“混过去”。
* verifier 不完美时，resampling 收益上限明显。([arXiv][3])

**争议**

* R1 风格收益到底来自真正 reasoning，还是更好地放大了 pretrained pass@k？([arXiv][28])
* GRPO 与 DPO 的等价/近等价关系削弱了“新 RL 算法”的新颖性。([arXiv][10])

**可突破点**

* 不再只问“答案对不对”，而问“哪一步让答案变对”。
* 用 interventional credit 把 RLVR、PRM、verification 统一起来。

---

### 3.2 D2 细粒度 credit assignment（PRM / GenRM / dense reward）

**现状**

* PRIME/PURE/CAPO/GenRM 都在尝试给 step-level reward；思路从 explicit PRM 到 generative verifier，再到 min-form credit。([arXiv][22])
* 社区共识已经形成：真正难题不是能不能训练，而是**credit 怎么给得 faithful**。([arXiv][29])

**瓶颈**

* 观察性 step score 容易把“看起来合理”的步骤误当成“真正必要”的步骤。
* Dense reward 往往受风格/长度/位置偏差影响。([arXiv][30])

**争议**

* summation vs min-form：到底该累计所有好步骤，还是只抓瓶颈步骤。([arXiv][31])
* generative verifier 的分数是否真比判别式更 faithful。([arXiv][30])

**可突破点**

* 用 counterfactual necessity 连接“sum-form”“min-form”“outcome-only”。
* 同时给出新训练信号和新评测，形成更强叙事。

---

### 3.3 D3 可靠性 / abstention / calibration

**现状**

* 多个 benchmark 开始显示 reasoning FT 与 honest uncertainty 存在冲突，尤其 AbstentionBench 的结论很醒目。([开放评论][4])
* 科学/临床 QA 的 calibration benchmark 开始变成熟。([arXiv][32])

**瓶颈**

* 现有 uncertainty 往往只看答案层面，不看 trace 是否“支撑”答案。
* 很多模型“思考越多，越自信地错”。([arXiv][5])

**争议**

* abstain 是否会牺牲有用回答率。
* reasoning traces 会不会被模型用来 rationalize，而不是 reveal cognition。([arXiv][33])

**可突破点**

* 把 abstention 与 trace necessity 绑定：如果没有必要步骤，就该拒答。
* 这是一个很强的新评测视角。

---

### 3.4 D4 机制解释：SAE / circuits / thought tracing

**现状**

* SAE 与 circuit tracing 已能对大模型内部 feature 与路径做较高分辨率分析。([变压器电路][8])
* Universal feature space 和 auto-interpretation 让跨模型比较成为可能。([arXiv][34])

**瓶颈**

* feature 很多，但和训练目标/能力提升脱节。
* 干预结果常难转化成可复用算法。([arXiv][7])

**争议**

* monosemantic feature 是否足够稳定。
* auto-interpretation 容易产生 post-hoc 故事。([arXiv][35])

**可突破点**

* 只把机制解释作为“验证主方法为何有效”的第二证据链，而不是主方法本身。
* 分析“高必要性步骤”是否对应稳定 feature/circuit 激活。

---

### 3.5 D5 动态 benchmark / contamination science

**现状**

* LiveBench/LiveCodeBench/MMLU-CF/HLE-Verified 说明 benchmark 本身正在成为一等研究对象。([开放评论][11])
* 诊断型评测如 RAGChecker 已显示，粗粒度最终分数遮蔽了关键 failure mode。([arXiv][36])

**瓶颈**

* 新 benchmark 若没有方法或机制支撑，很难冲 best paper。
* 数据维护成本高。([开放评论][11])

**争议**

* live benchmark 的时效性与 reproducibility 的张力。
* benchmark 更新频繁会影响纵向可比性。([开放评论][11])

**可突破点**

* 给主方法配一个“counterfactual trace benchmark”，把评测做成理论支点，而不是孤立数据集。

---

### 3.6 D6 长上下文 / RAG / memory-grounded reasoning

**现状**

* LongBench v2、RULER、Self-Route、MemoRAG 表明“更长上下文 ≠ 更好 grounding”。([arXiv][16])
* 最新工作已经把长上下文 RL 的核心瓶颈归结为 sparse final reward 无法给 grounding credit。([arXiv][18])

**瓶颈**

* 证据使用是 latent 的；很难知道模型到底用了哪一段上下文。
* 成本高，失败模式多。([arXiv][37])

**争议**

* RAG 和 long context 谁更优不是静态结论，而是 task-dependent。([arXiv][17])

**可突破点**

* 把 necessity 定义到“引用的证据片段/步骤”上，形成 grounding-aware credit。
* 这能成为主推 idea 的 Plan B。

---

### 3.7 D7 Agents + agent safety

**现状**

* Search-R1 / WebSailor / ReSum 等把 web research agent 推到新高度；同时 WASP / OS-Harm / visual prompt injection 暴露巨大安全面。([arXiv][24])

**瓶颈**

* 环境重、变动快、复现实验昂贵。
* 结果常被工程细节主导。([arXiv][15])

**争议**

* 单任务成功率是否代表真实 agent intelligence。
* 安全攻击 benchmark 是否覆盖真实攻击面。([arXiv][38])

**可突破点**

* 做子问题：例如“web evidence necessity”而非全栈 agent。

---

### 3.8 D8 Formal reasoning / theorem proving / verified code

**现状**

* theorem proving 与 verified code 是最自然的 verifiable RL 试验田。([arXiv][19])
* 多个 prover 系统已经证明：层次化分解与自博弈有明显收益。([arXiv][19])

**瓶颈**

* proof assistant 生态门槛高；训练与评测碎片化。
* 若只在 formal math 上有效，外部影响力可能受限。([arXiv][20])

**争议**

* 模型是在学证明，还是在学 tactic prior。
* proof trace faithfulness 尚未解决。([arXiv][39])

**可突破点**

* 作为第二域验证非常强：若主方法在 math 和 theorem proving 都成立，可信度会大增。

## 4. 候选课题（3–5 个）

### C1. **Counterfactual Necessity Training for Reasoning**

* **核心问题定义**：在可验证推理任务上，怎样识别并强化“真正让答案成立的必要步骤”，而不是“看起来合理但可有可无的步骤”？
* **关键 gap**：现有 RLVR/PRM 大多基于 outcome 或观察性 step score，缺 interventional credit。
* **best-paper 支点**：**新机制 + 新评测 + 轻理论统一**。
* **为什么现在做合适**：R1 后所有人都在堆 post-training；这时提出一个统一原则比再做 loss tweak 更有叙事力。([arXiv][1])

### C2. **Trace-Grounded Abstention for Reasoning Models**

* **核心问题定义**：模型应在“没有必要推理证据”时拒答，而不是输出漂亮但站不住的 chain-of-thought。
* **关键 gap**：abstention 研究大多停留在 answer-level confidence，没有 trace-level support notion。
* **best-paper 支点**：**新问题定义 + 新评测**。
* **为什么现在做合适**：AbstentionBench 刚把问题抬到台前，社区正缺“为什么 reasoning FT 会伤 abstention”的机制解释。([开放评论][4])

### C3. **Verifiable Grounding Credit for Long-Context RL**

* **核心问题定义**：在长上下文推理中，怎么把 reward 分配给真正被用到的证据片段与中间步骤？
* **关键 gap**：final-answer reward 无法训练 grounding；长上下文 RL 缺 dense、faithful、可验证的中间信号。
* **best-paper 支点**：**新机制 + 新应用场景**。
* **为什么现在做合适**：已有工作明确指出 sparse reward 对长上下文 grounding 不够；这是正在形成的新主线。([arXiv][18])

### C4. **Mechanistic Predictors of Spurious Reasoning**

* **核心问题定义**：是否能用 SAE/circuit-level 信号预测哪些 reasoning steps 是 spurious、哪些会对正确性真正起作用？
* **关键 gap**：机制解释与训练目标脱节，缺可操作变量。
* **best-paper 支点**：**机制解释 + 因果干预**。
* **为什么现在做合适**：SAE/circuit 工具成熟到可以做 task-grounded 验证，但需要与一个更强主问题绑定。([变压器电路][8])

### C5. **Counterfactual Trace Benchmark**

* **核心问题定义**：构建一个动态、可验证、反事实式 benchmark，专门测“过程是否必要/faithful”。
* **关键 gap**：当前 benchmark 多测最终正确率，无法区分“对答案有帮助的步骤”和“装饰性步骤”。
* **best-paper 支点**：**新评测**。
* **为什么现在做合适**：LiveBench/HLE-Verified 已证明“benchmark 本身的缺口”能改变社区结论。([开放评论][11])

---

### 4.1 在选主推 idea 前，我先回答 4 个关键问题

#### Q1. 最接近的 3 篇 / 3 类工作是谁？我们多了什么不可替代的东西？

1. **Outcome-only RLVR / GRPO 类**：DeepSeek-R1、DAPO、Dr.GRPO、R1-Zero-like training。

   * 它们优化“最终是否答对”；我们优化“哪一步让它答对”。([arXiv][2])
2. **Step-level dense reward / PRM 类**：PRIME、PURE、Retrieval-PRM。

   * 它们主要是观察性或聚合式 step reward；我们用 **counterfactual intervention** 定义 necessity/harmfulness。([arXiv][22])
3. **Generative verifier / verifier-guided inference 类**：GenRM、Heimdall、adaptive verification。

   * 它们多把 verifier 当 judge；我们把 verifier 当 **interventional oracle**，生成训练 credit 和新评测。([arXiv][30])

#### Q2. 这个贡献属于哪类？

* **组合型贡献**：**新机制 + 新评测 + 轻理论**。
* 强点在于：不是“把 RL objective 从 A 改成 B”，而是改了监督对象本身——从“成功轨迹”变成“必要步骤”。

#### Q3. 在我们算力约束下，最小闭环是什么？

* **2–3 周闭环**：

  * 选 3k 个可验证 math 问题。
  * 每题采样 4 条轨迹。
  * 对每条轨迹只挖 2–3 个高信息量步骤做 drop/swap/repair counterfactual。
  * 训练 7B LoRA step preference 模型。
  * 在 MATH-500 / AIME24 上看三个信号：accuracy、Counterfactual Trace Faithfulness、compute-normalized accuracy。
* 若这三个信号一起变好，再扩大。

#### Q4. 这个方向的 best-paper 级叙事是什么？

* **一句话叙事**：
  **“推理后训练真正缺的不是更多成功轨迹，而是知道哪一步让成功发生；一旦把监督从‘答对’改成‘必要步骤’，训练、验证和评测会被统一起来。”**

## 5. 主推 idea（完整研究蓝图）

# 主推：**Counterfactual Necessity Training (CNT)**

副产品 benchmark：**CounterTrace**

### 5.1 一句话主张（thesis claim）

**在可验证推理任务上，优化“反事实必要步骤”比优化“最终正确答案”或“观察性步骤分数”更能提升准确率、faithfulness 与测试时计算效率。**

---

### 5.2 创新点列表（novelty checklist）

#### 与最接近的 3 类工作逐项对比

| 维度               | Outcome-only RLVR / GRPO | PRM / PRIME / PURE | GenRM / Heimdall / verifier TTS | **CNT（我们）**                       |
| ---------------- | ------------------------ | ------------------ | ------------------------------- | --------------------------------- |
| 监督对象             | 最终结果                     | 观察性 step score     | 轨迹/答案的 verifier score           | **interventional step necessity** |
| 是否能区分“必要”与“看似合理” | 弱                        | 中等                 | 中等                              | **强**                             |
| 是否提供新评测          | 弱                        | 弱                  | 弱                               | **强：CounterTrace / CTF / MSTL**   |
| 是否有机制解释空间        | 中                        | 中                  | 中                               | **强：necessity 统一视角**              |
| 是否依赖昂贵在线 RL      | 通常需要                     | 不一定                | 不一定                             | **先离线可做，后在线可选**                   |

**至少 5 条 novelty：**

1. **新监督对象**：首次把 reasoning step 的训练信号定义为 **counterfactual necessity / harmfulness / stability**，而不是 purely observational score。
2. **新统一原则**：把 outcome-only、sum-form PRM、min-form PRM 统一为 necessity 估计的不同近似。
3. **新评测镜头**：提出 **Counterfactual Trace Faithfulness (CTF)**、**Minimal Sufficient Trace Length (MSTL)**、**Step-Lift AUC**。
4. **新训练管线**：离线挖掘 counterfactual step pairs，训练 pairwise step policy；再选配轻量 2-GRPO refinement。
5. **可证伪机制**：若 necessity 真是关键，那么“高 necessity 步骤删除后 verifier 成功率应显著下降；仅表述改写不应显著下降”。
6. **compute 叙事**：比起盲目 sample more，CNT 应该在相同 test-time token budget 下更高效。
7. **跨域性**：math → code/formal 可迁移，因为定义只要求“结果可验证”。

---

### 5.3 核心洞察

* **这不是工程堆砌**。它改变的是“我们认为 reasoning supervision 应该对准什么”。
* 现有方法默认两种极端：

  * **uniform credit**：所有正确轨迹 token 都有用；
  * **bottleneck credit**：只有最差一步决定成败。
* 我们的观点是：真正该学的是 **局部干预下会改变成败的步骤**。这是一种更接近因果意义的 credit。
* 所以它天然带来：

  * 更稳的 step reward；
  * 更能解释为什么某些后训练有效；
  * 更自然的 faithful evaluation。

---

### 5.4 方法定义

#### 5.4.1 数据对象

每个样本为：

* 问题 (x)
* 一条推理轨迹 (\tau = (s_1,\dots,s_T))
* 结果验证器 (V(x,\tau)\in [0,1]) 或 ({0,1})

#### 5.4.2 反事实步骤操作

对第 (t) 步做局部操作：

1. **Drop**：删除该步并让模型续写。
2. **Swap**：替换成另一条轨迹在同前缀下的步骤。
3. **Repair**：对该步做最小修复，再续写。
4. **Paraphrase-control**：仅改写表述，不改语义。

记编辑后的完成轨迹为 (\tilde\tau^{(o)}_t)。

#### 5.4.3 necessity / harmfulness / stability

定义：
[
n_t = V(x,\tau)-\mathbb{E}_{o\in{\text{drop,swap}}}[V(x,\tilde\tau^{(o)}_t)]
]

[
h_t = \mathbb{E}[V(x,\tilde\tau^{(\text{repair})}_t)]-V(x,\tau)
]

[
u_t = 1-\left|V(x,\tau)-\mathbb{E}[V(x,\tilde\tau^{(\text{para})}_t)]\right|
]

其中：

* (n_t)：这一步有多“必要”
* (h_t)：这一步有多“有害/可修复”
* (u_t)：这一步的 credit 是否稳定，而非格式噪声

合成 step credit：
[
c_t = u_t \cdot (n_t + \lambda_h h_t)
]

#### 5.4.4 训练样本构造

在同一前缀 (p_t=(x,s_{<t})) 下，构造：

* 正样本步骤 (s_t^+)：高 (c_t)
* 负样本步骤 (s_t^-)：低 (c_t) 或 harmful step

得到加权 pair ((p_t,s_t^+,s_t^-,w_t))，其中 (w_t=|c_t^+ - c_t^-|)。

#### 5.4.5 训练目标

**Step pairwise loss**（避免长度偏差，用 step 平均 logprob）：
[
L_{\text{step}}=\sum_i w_i \log\left(1+\exp\left[-\beta\left(\bar{\log\pi_\theta}(s_i^+|p_i)-\bar{\log\pi_\theta}(s_i^-|p_i)-m\right)\right]\right)
]

总损失：
[
L = \lambda_{\text{sft}}L_{\text{SFT}}+\lambda_{\text{step}}L_{\text{step}}+\lambda_{\text{crit}}L_{\text{credit-reg}}
]

其中 (L_{\text{credit-reg}}) 是让一个小 critic 预测 (c_t) 的回归损失，供后续在线阶段使用。

#### 5.4.6 可选在线 refinement

若离线成功，再做轻量 2-GRPO：
[
R(\tau)=\lambda_o V(x,\tau)+\lambda_c \sum_t \hat c_t - \lambda_l \text{len}(\tau)
]

这里 (\hat c_t) 是 critic 预测的 necessity credit。

### 5.5 伪代码（伪代码级）

```text
Algorithm 1: Counterfactual Necessity Training (CNT)

Input:
  dataset D = {(x, y*)}, solver π0, verifier V
  K trajectories/problem, M edited variants/selected step

1. For each problem x in D:
2.   Sample K reasoning trajectories τ1...τK ~ π0(.|x)
3.   Segment each τ into steps s1...sT
4.   Select informative steps S (entropy / disagreement / verifier-margin based)
5.   For each selected step st:
6.       Create edited versions:
           drop(st), swap(st), repair(st), paraphrase(st)
7.       Complete edited prefix with solver/editor
8.       Evaluate each completed trace with verifier V
9.       Compute n_t, h_t, u_t, and c_t
10.      Build weighted local preference pairs under same prefix
11. Train solver πθ with sequence SFT + weighted step pair loss
12. Optionally train critic ĉ_t ≈ c_t
13. Optionally run short online RL stage with terminal verifier reward + dense necessity reward
14. Evaluate on:
      - final accuracy
      - Counterfactual Trace Faithfulness
      - compute-normalized accuracy
      - OOD transfer / robustness
Output: πθ and CounterTrace benchmark
```

### 5.6 关键超参、训练细节、复杂度

#### 建议默认配置

* **backbone**：Qwen2.5-Math-7B 或同级 reasoning 7B；第二阶段再上 14B 验证趋势。
* **采样**：每题 (K=4) 条轨迹。
* **步骤选择**：每轨迹选 2–3 个高信息量步骤。
* **编辑数**：每步 3–4 个反事实编辑。
* **loss 超参**：

  * (\lambda_{\text{sft}}=1.0)
  * (\lambda_{\text{step}}=0.5 \sim 2.0)
  * (\lambda_{\text{crit}}=0.1)
  * (\beta=0.1 \sim 0.5)
  * margin (m=0) 或小正数
  * (\lambda_h=0.5) 起调
* **训练方式**：LoRA/QLoRA 起步；成功后可做 full FT 小规模复验。
* **推理**：greedy + self-consistency 各一条线；必要时用 verifier rerank。

#### 复杂度

* 反事实挖掘成本约为：
  [
  O(N \cdot K \cdot T_s \cdot M)
  ]
  其中 (T_s) 是被选择的步骤数，不是全步骤。
* 这是可控的，因为：

  * 只挖 informative steps；
  * 用 prefix cache / vLLM；
  * 可先用便宜 verifier 粗筛，再用强 verifier 精筛。

### 5.7 至少 3 个强基线

1. **最难打败的 baseline**：**Dr.GRPO / 2-GRPO 风格 outcome-only RLVR**

   * 因为它是当前最接近主流 recipe 的强精调基线。([arXiv][40])
2. **最接近 dense-reward baseline**：**PRIME**

   * 代表“implicit process reward”路线。([arXiv][22])
3. **最接近 fine-grained credit baseline**：**PURE / Stop Summation**

   * 代表“min-form credit”路线。([arXiv][31])
4. **补充 baseline**：**CAPO / GenRM / Heimdall-guided rerank**

   * 用来证明不是“你只是用了更强 verifier”。([arXiv][41])

### 5.8 关键实验（minimum viable experiments，至少 6 个）

#### E1. Synthetic necessity recovery

* **做什么**：构建有 ground-truth indispensable step 的合成算术/算法推理任务。
* **指标**：step importance rank correlation / AUC。
* **预测**：CNT 的 (c_t) 明显优于 entropy、token surprise、PRM score、GenRM score。
* **证伪性**：如果不优于这些基线，就说明 necessity 估计并没学到新东西。

#### E2. 主任务准确率

* **做什么**：在 MATH-500、AIME24、AMC23 或同级可验证题集上比较。
* **指标**：pass@1、majority vote、verified accuracy。
* **预测**：在相同 7B 规模和相似训练预算下，CNT 在 hardest split 上优于 Dr.GRPO / PRIME / PURE。

#### E3. Counterfactual Trace Faithfulness（CTF）

* **做什么**：删除 top-k 高 credit steps、删除随机 steps、仅改写表述三组比较。
* **指标**：verifier success drop、CTF、MSTL。
* **预测**：CNT 学到的 top-k 步骤删除后性能降幅最大，而 paraphrase-control 降幅最小。

#### E4. Reward hacking / verbosity stress test

* **做什么**：向轨迹中加入冗长但语义空洞步骤、answer-first steps、style-shift steps。
* **指标**：accuracy、CTF、被 verifier 欺骗率。
* **预测**：summation 风格 PRM 更易被 verbosity 骗；CNT 更稳。

#### E5. Compute-efficiency frontier

* **做什么**：固定 test-time token budget，比 1-shot、self-consistency、verifier rerank。
* **指标**：accuracy vs tokens / verifier calls。
* **预测**：CNT 在相同预算下更高，说明学到的是更“有效”的 reasoning，而非更长 reasoning。

#### E6. Ablation

* **做什么**：分别去掉 drop、swap、repair、paraphrase-control；去掉 critic；去掉 length normalization。
* **预测**：

  * drop+repair+control 最强；
  * 去掉 control 后稳定性变差；
  * 去掉 length normalization 会重新出现长步骤偏好。

#### E7. 第二域验证（可选但很值）

* **做什么**：放到 code reasoning 或 theorem-lite 任务。
* **预测**：趋势仍在，说明不是 math-only trick。

#### E8. 机制验证（加分项）

* **做什么**：用 SAE/circuit tracing 看高 necessity 步骤是否对应更稳定、可复现的 feature 激活。
* **预测**：高 necessity 步骤会关联更稀疏、更一致的中层 feature 子图。

---

### 5.9 两个预期会失败的设置

1. **高冗余、多种等价证明都成立的任务**

   * necessity 变得分散，单步 credit 不再尖锐。
   * 失败也有价值：说明方法适用边界是“存在少量关键步骤”的任务。

2. **弱 verifier / 开放式主观任务**

   * counterfactual editing 的 outcome 不稳定，credit 不可识别。
   * 失败也有价值：说明本方法依赖 verifiability，不适合无金标准开放生成。

---

### 5.10 潜在风险与备选路线（Plan B / C）

#### Plan B：转向长上下文 grounding

* 把“step necessity”替换成“evidence necessity”。
* 对引用的文段做删除/替换/修复，训练 grounding-aware reward。
* 这与 2026 的 verifiable grounding 线高度契合。([arXiv][18])

#### Plan C：转向 abstention

* 把 CTF 作为拒答依据。
* 规则：若高 credit step 不稳定，模型应该 abstain。
* 能形成“why reasoning FT hurts abstention”的机制解释。([开放评论][4])

---

### 5.11 如果结果不如预期，怎么迭代（debug roadmap）

* **症状 A：accuracy 没涨，CTF 也没涨**
  → 先查 counterfactual pipeline 噪声；看 paraphrase-control 是否也大幅改变 verifier。
  → 若是，收紧编辑 prompt、只保留高一致性样本。

* **症状 B：CTF 涨了，accuracy 不涨**
  → 说明学到 faithful 但保守的步骤偏好。
  → 加小规模在线 RL 或 inference-time rerank。

* **症状 C：模型变短、过早作答**
  → 增大 sequence SFT 比重；对过短轨迹加 coverage penalty。

* **症状 D：仍然偏爱冗长步骤**
  → 检查是否忘了 step 平均 logprob；加强 length-adversarial negative。

* **症状 E：迁移差**
  → 用 retrieval-augmented step memory 或多任务混合，再测第二域。

## 7. 写作框架（标题 / 摘要 / 图表 / related work）

### 7.1 标题候选 × 5

1. **Necessity, Not Plausibility: Counterfactual Credit Assignment for Verifiable Reasoning**
2. **Which Step Made It Right? Counterfactual Necessity Training for Reasoning Language Models**
3. **From Right Answers to Necessary Steps: Interventional Supervision for Reasoning Models**
4. **Beyond Outcome Rewards: Counterfactual Step Necessity for Training and Evaluating Reasoning LMs**
5. **CounterTrace: Measuring and Optimizing Necessary Reasoning Steps in Language Models**

---

### 7.2 摘要骨架

* **问题**：现有 reasoning post-training 优化最终答案或观察性过程分数，无法区分必要步骤与装饰性步骤。
* **方法**：提出 CNT，用 drop/swap/repair/paraphrase 反事实干预估计 step necessity，并据此训练 reasoning model。
* **评测**：提出 CounterTrace、CTF、MSTL，专门测 trace faithfulness。
* **结果**：在可验证 math（及可选 code/formal）上，CNT 提升 accuracy、faithfulness 与 compute efficiency。
* **机制**：证明或实证显示，uniform reward 与 bottleneck reward 是 necessity 原则的粗糙近似。
* **意义**：把 post-training、verification、evaluation 统一到“必要步骤”这一原则下。

---

### 7.3 贡献点 bullet

* 提出 **counterfactual necessity** 作为 reasoning credit 的新原则。
* 提出 **CNT**，一种离线可训练、在线可扩展的 step-level training 方法。
* 提出 **CounterTrace / CTF / MSTL**，测量 reasoning trace 是否真正必要。
* 证明或实证统一了 outcome reward、sum-form PRM、min-form PRM。
* 在相同中等算力下，实现更强 accuracy-faithfulness-compute 三重改进。

## 最终判断

### 我最推荐你们押注的方向

**C1 / CNT：Counterfactual Necessity Training for Reasoning**

### 原因

* 它踩在 **最热主线** 上，但不落入“再造一个 RL 配方”的红海。([arXiv][1])
* 它同时满足你的偏好：

  * **不是堆预训练规模**
  * **算力可控**
  * **有强机制解释空间**
  * **有清晰可证伪预测**
  * **可以做出新评测 moat**
* 它的 best-paper 叙事清楚：
  **“我们找到的不是更强 trick，而是 reasoning post-training 真正应该对齐的监督对象。”**
