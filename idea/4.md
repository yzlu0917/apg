## 1. 方向地图（含主题覆盖）

### 1.1 Step A：广撒网（覆盖 10/10 主题；每个主题“主线/痛点/重要性”）

1. **推理与验证（reasoning/verifier/self-correction）**

* 主线：把“更强推理”从训练期搬到**测试期（test-time compute）**，用 **verifier / PRM / search** 做候选生成与验证的闭环。
* 痛点：verifier 误差会被 **Best-of-N / search 放大（reward hacking）**，而且不同粒度验证（step vs final vs adaptive）缺少统一理论与稳定评测。
* 重要性：这是当前“推理能力跃迁”的核心路径之一，且最容易做出“机制解释 + 可证伪预测”。

2. **可靠性与对齐（robustness/honesty/calibration/uncertainty）**

* 主线：把 LLM 的“自信”变成可度量、可控的**不确定性**与**选择性回答（abstain）**；将“诚实/不撒谎”作为可训练与可评测对象。
* 痛点：校准在**分布漂移**下失效；“高准确≠高可靠”，而很多方法只在静态基准有效。
* 重要性：对 agent、RAG、judge、RLHF 都是底座；也是最容易引入**统计保证/理论**的入口。

3. **表征与机制解释（mechanistic interpretability / representation learning）**

* 主线：SAE/稀疏特征、因果干预、可介入特征，试图把“模型为何有效/为何失败”变成可复现实证。
* 痛点：特征**非唯一**、解释**可伪造**、评价标准不统一；很多工作停留在“可视化”而非“可证伪机制”。
* 重要性：若能把机制解释与训练/推理改进闭环，会很接近 best-paper 叙事。

4. **训练与优化新范式（RLHF/RLAIF alternatives, off-policy RL, direct preference）**

* 主线：DPO/变体、博弈视角对齐、off-policy RLVR，提高数据效率与稳定性。
* 痛点：偏好噪声、non-stationarity、credit assignment、以及“verifiable→unverifiable”迁移。
* 重要性：对齐方法仍处快速演化期，且存在可做的理论空白。

5. **数据合成与评测（synthetic data, evaluation, contamination, benchmark design）**

* 主线：用合成数据做训练与评测，用 LLM-as-judge 扩展规模，同时用动态基准/水印/因果实验对抗污染。
* 痛点：合成数据质量评估困难；judge 有系统偏差；污染让 leaderboard 变得不可信。
* 重要性：这是“研究可验证/可复现”的护城河来源之一（新评测往往带来大影响）。

6. **高效学习（PEFT/low-data/modularity/compositionality）**

* 主线：LoRA/adapter/模块化组合，让“学新技能”更快更可控；用组合泛化检验真正能力。
* 痛点：模块如何组合、如何避免遗忘、如何在新任务上可解释地迁移仍不清晰。
* 重要性：你们算力足够做系统 ablation，但不必堆预训练。

7. **多智能体/工具使用（agents, tool use, planning, search）**

* 主线：把 LLM 放进环境，通过工具调用、规划、长时任务完成度来衡量“智能体能力”。
* 痛点：评测标准混乱、长时任务不可控、工具链脆弱、容易 prompt injection。
* 重要性：顶会关注度极高，但 best-paper 往往需要“可解释机制 + 强评测”。

8. **检索与记忆（RAG/long-context/memory systems）**

* 主线：RAG vs long-context 的系统比较；记忆系统（写入/检索/遗忘/一致性）与鲁棒性。
* 痛点：检索噪声、上下文选择、长上下文 KV 成本、记忆注入攻击。
* 重要性：落地价值强，且很适合做“新评测 + 机制分析”。

9. **结构化生成/程序归纳（code/formal methods/theorem proving）**

* 主线：用可验证的形式系统（Lean/SMT/单元测试）把“正确性”做成硬信号，驱动训练与评测。
* 痛点：数据稀缺、搜索空间大、从数学到代码验证迁移难。
* 重要性：最容易做出“可验证、可复现”的硬贡献，也利于理论化。

10. **安全与红队（jailbreak/adversarial robustness）**

* 主线：自动化 jailbreak、prompt injection 防御、以及对防御的系统评测。
* 痛点：攻防军备竞赛；“防住某个基准”不等于真实安全。
* 重要性：顶会持续高热，若能提出**新威胁模型/新评测镜头**很可能高影响。

### 3.1 D1 Test-time compute scaling + verifier/search

**代表性论文（近 6–18 个月优先）**

* **pass@N 与 CE 训练失配；“限制置信度”的训练损失**（NeurIPS’25）([NeurIPS][1])
* **Best-of-N 大 N 会 reward hacking；提出 InferenceTimePessimism 且给理论**（ICML’25）([ICML][2])
* **验证粒度 g：VG-Search 统一 beam 与 Best-of-N；自适应粒度更省算**（NeurIPS’25）([NeurIPS][3])
* **Reward Reasoning Models：reward model 自己“推理后给分”，并能用 test-time compute**（NeurIPS’25）([OpenReview][4])
* **AgentTTS：agent 领域的 test-time scaling**（NeurIPS’25）([NeurIPS][42])
* **PRM + MCTS 组合（LE-MCTS）**（arXiv’24/25）([arXiv][43])
* **ReVISE：测试时自验证/自修正来驱动 RL（ICML’25）**([ICML][44])
* **Latent reasoning 的 test-time scaling（NeurIPS’25）**([NeurIPS][45])

**现状范式**

* 生成候选（sampling/beam/search）→ verifier 打分（RM/PRM/规则/执行器）→ 选最优或继续搜索。
* 研究正从“堆 N”走向：**训练-推理 co-design**（pass@N 训练失配）、**验证粒度/成本**、**不确定性驱动的选择**。

**主要瓶颈 / failure cases（FMM）**

* **reward hacking**：N 增大时，极端样本更容易骗过 verifier（ICML’25 明确分析）([ICML][2])
* verifier **未校准**：分数不可比，跨任务/跨提示漂移。
* “验证粒度”选错：step-level 太贵；final-only 太脆弱（VG-Search 针对这一点系统化）([NeurIPS][3])

**争议点 / negative results**

* “更多 test-time compute 总更好？”：Best-of-N 在大 N 下会变差（理论+实证）。([ICML][2])
* “reward model 推理就更可靠？”：RRMs 虽提升，但依旧可能被分布漂移/对抗样本破坏。([OpenReview][4])

**可突破点（gaps）**

* **(NEL + SCL)**：缺少统一的 **Risk–Compute** 曲线与“可置信选择”的评测协议（不仅 accuracy）。
* **(AST)**：很多方法隐含假设“verifier 错误独立/同分布”，在搜索与选择后不成立。
* **(UP)**：把“test-time search + verifier + uncertainty”统一成**统计选择/假设检验**问题（见主推 idea）。

### 3.2 D2 不确定性/校准/选择性回答（abstain）/诚实

**代表性论文**

* NeurIPS’25：系统比较 TPU/NVU/LVU，指出“高准确≠可靠不确定性”([NeurIPS][5])
* Conformal linguistic calibration（NeurIPS’25）([NeurIPS][6])
* DS-CP：domain shift 下 reweight calibration 保持 coverage（arXiv’25）([arXiv][7])
* “何时应该更不具体”：选择性输出减少错误断言（arXiv’26）([arXiv][8])
* SelectLLM：用选择机制提升输出可靠性（NeurIPS’25）([NeurIPS][46])
* “Confident failures”：自一致性也可能高自信地错（NeurIPS’25）([NeurIPS][47])
* Incentivizing truthful LMs via peer elicitation（NeurIPS’25）([NeurIPS][48])
* Evaluating honesty in LLMs（OpenReview）([OpenReview][49])

**现状范式**

* 以 **selective prediction**（答/不答）为中心：校准置信度→设阈值→控制风险（但多数只做 marginal）。
* conformal 方法开始进入 LLM，强调**有限样本/分布无关**的 coverage/risk。

**瓶颈（FMM + AST）**

* **分布漂移**会让标准 conformal under-cover（DS-CP 专门指出并修复）([arXiv][7])
* 许多“置信度 proxy”（logprob、self-consistency）与真实错误排序不一致。([NeurIPS][5])

**争议点**

* “校准好就能提升决策质量？”：NeurIPS’25 指出“好校准不一定带来好 error ranking”。([NeurIPS][5])

**可突破点（NEL + UP）**

* 将不确定性从“输出层面”升级到“**搜索/选择层面**”：对 **best-of-N / PRM search** 的最终选择提供**可置信保证**（主推）。
* 在 **domain shift + selection bias**（被选择的样本不是 IID）下给出可复现保证。

### 3.3 D3 LLM-as-judge：统计纠偏 + 置信区间 + 选择性裁决

**代表性论文**

* “正确报告 LLM-as-judge”：用 human-calibration 数据纠偏 + 置信区间（arXiv’25）([arXiv][9])
* SCOPE：选择性 pairwise judging + 有限样本风险约束（arXiv’26）([arXiv][10])
* ICML’25：unbiased evaluation 的理论化视角（Agents-as-an-Evaluator 偏差）([ICML][11])
* Agent-as-a-Judge（ICML’25）([ICML][12])
* Valid inference with imperfect synthetic data（NeurIPS’25）([NeurIPS][50])
* DATE-LM：数据归因评测基准（NeurIPS’25）([NeurIPS][51])

**现状范式**

* judge 变成训练/评测基础设施，但社区正在补齐：**bias correction、CI、abstain**。

**瓶颈（AST）**

* 大量论文仍把 judge 当“近似人类”，忽略 sensitivity/specificity 与分布漂移。([arXiv][9])
* 选择性判断（只评一部分）会破坏标准统计有效性（需要 selective inference/conformal）。([arXiv][10])

**可突破点（UP + NEL）**

* 把“judge 的可信度控制”迁移到“**verifier 选择正确解**”的可信度控制：同一套统计语言统一两个领域（评测与推理）。

### 3.4 D4 Mechanistic interpretability：SAE/因果干预/可证伪解释

**代表性论文**

* NeurIPS’25：用“近邻负例”去**证伪**SAE 特征解释，并提出结构化解释格式([NeurIPS][13])
* EMNLP’25 RouteSAE：更高效/更好路由的 SAE 训练或结构设计([ACL Anthology][14])
* “因果是解释可泛化关键”（arXiv’26）([arXiv][15])
* ICML’25：global-level mechanistic interpretability（从局部到全局）([ICML][16])
* 2026：可介入、可解释特征识别（arXiv’26）([arXiv][52])
* NeurIPS’25：ViT 的可解释性（residual replacement）([NeurIPS][53])

**瓶颈（FMM）**

* **解释不唯一**、容易被 prompt/数据选择“伪造”。
* 许多评估不能回答：解释是否支持**反事实干预**与**可迁移预测**。

**可突破点（NEL + AST + UP）**

* 需要把“解释质量”变成：

  * 可证伪预测（干预某特征→某类错误上升/下降）；
  * 对下游（verifier/校准）带来可量化增益。

### 3.5 D5 Direct alignment：DPO 鲁棒/博弈/多目标

**代表性论文**

* DPO-PRO：对噪声偏好更鲁棒（NeurIPS’25）([NeurIPS][17])
* Risk-aware DPO（NeurIPS’25）([NeurIPS][18])
* 样本调度提升 DPO（NeurIPS’25）([NeurIPS][54])

## 4. 候选课题（3–5 个最可能冲击 best paper）

### 课题 A（主推候选）：**Risk-Controlled Test-Time Search：让 verifier-guided scaling “可置信且不 reward-hack”**

* 核心问题：test-time compute scaling 很强，但大 N/深 search 会放大 verifier 错误；能否对“最终选出的答案”给出**可验证的风险上界**？
* 关键 gap：现有工作要么指出 reward hacking（并给特定算法如 pessimism）([ICML][2])，要么做 conformal/选择性判断但不处理“多候选选择偏差”([arXiv][10])。
* best-paper 支点：**把 test-time selection 形式化为 selective inference/conformal**，给**有限样本风险控制** + 新评测镜头（Risk–Coverage–Compute）。
* timing：NeurIPS’25 已出现 TTS、验证粒度、reward reasoning models 等“拼图”([NeurIPS][1])，2026 初出现 selective conformal 工具（PEMI/SCOPE）([arXiv][10])，正是整合窗口。

### 课题 B：**Selection Bias in LLM Reliability：把“校准”从 IID 扩展到“被挑选出来的样本”**

* 核心问题：实际系统只在“模型看起来很自信/被 search 挑出来”的样本上输出；标准校准与 conformal 在选择后会失效。([arXiv][68])
* gap：现有 DS-CP 处理 domain shift，但对 selection-conditional coverage 的系统落地不足。([arXiv][7])
* 支点：新理论（SCC）+ 新实证（search/agent 场景）+ 可复现协议。

### 课题 C：**可信 LLM-as-judge → 可信训练信号：把纠偏/CI 引入 alignment pipeline**

* 核心问题：judge 常被用于训练（偏好数据/奖励）；如果评估都偏，训练信号也偏。
* gap：已有“纠偏+CI”主要用于报告评测分数([arXiv][9])，很少形成闭环“纠偏 judge → 更稳训练”。
* 支点：把统计纠偏融入 DPO/RLHF 训练目标，给可证伪预测（更稳、跨分布更可靠）。

### 课题 D：**Mechanistic Verifier Hacking：用可介入特征解释“为什么大 N 会骗过 verifier”**

* 核心问题：reward hacking 不是黑箱；能否用 SAE/因果干预定位“骗分特征”，并据此设计防 hack 的 verifier？([NeurIPS][13])
* gap：机制解释与 test-time scaling 仍相对割裂。
* 支点：机制+反事实干预+改进方法（有科学味道，但风险较高）。

### 课题 E：**Memory 系统的风险受控：在 memory injection/poisoning 下保证“可置信记忆调用”**

* 核心问题：记忆与 RAG 引入新攻击面；如何在长链路下控制错误/攻击成功率？([NeurIPS][65])
* gap：多数工作给攻击或工程性防御，缺少统一风险报告与理论化。
* 支点：新评测镜头 + 风险控制策略（与 A 可共享方法论）。

### 5.2 先回答我必须回答的关键问题（对外可见）

**Q1：最接近的 3 篇/3 类工作是谁？我们多了什么不可替代？**

1. **Inference-time alignment / Best-of-N 理论化 + InferenceTimePessimism**（ICML’25）([ICML][2])

   * 它给出“为何 Best-of-N 大 N 会崩”的理论与一种 pessimism 算法。
   * 我们不可替代：**把“可信”从算法技巧升级为“可验证风险控制”**（有限样本 guarantee），并扩展到多种 search/验证粒度/可选择输出协议，而不是只给一个算法。

2. **选择性 conformal 用于 LLM judge（SCOPE）**（arXiv’26）([arXiv][10])

   * 它解决“judge 何时可信”的风险控制。
   * 我们不可替代：把同一统计语言迁移到**“verifier 选择正确解”**这一更核心的推理环节，并处理**多候选选择偏差**（judge 论文通常是对固定样本打分，而我们是从很多候选里挑一个输出）。

3. **选择后有效性（selective conformal / PEMI）**（arXiv’26）([arXiv][68]) + **domain shift 下 conformal（DS-CP）**（arXiv’25）([arXiv][7])

   * 它们给出工具，但没有把问题“落”到 test-time search 的关键路径上。
   * 我们不可替代：**把 PEMI/DS-CP 变成可复现的 LLM 推理系统组件**，并给出新的统一评测曲线（Risk–Coverage–Compute）。

**Q2：贡献属于新机制/新理论/新评测/新问题定义中的哪一种？为什么足够强？**

* 组合拳：

  * **新理论**：选择偏差（从 N 个候选里挑一个）下的风险控制（SCC 风格保证）。([arXiv][68])
  * **新机制解释**：解释 reward hacking 的本质是“选择放大 verifier 误差”，并用风险控制把“compute 变强”与“误差放大”解耦。([ICML][2])
  * **新评测镜头**：Risk–Coverage–Compute（+ CI），替代只看 accuracy/pass@k。

**Q3：算力约束下最小闭环是什么？（2–3 周信号）**

* 用开源模型 + 可验证任务（math/code）做：

  1. 训练一个简单 verifier（或直接用现成 RM）；
  2. 做 Best-of-N 在大 N 下的 degrade 复现；
  3. 上一个“朴素 Bonferroni conformal 风险控制”版本，看是否同时：

     * 控制住“被认证正确但实际错误”的比例；
     * 缓解大 N 的 degrade。
* 若 2–3 周看不到：要么 verifier 太弱/校准失败，要么任务选择不合适（见 debug 树）。

**Q4：best paper 级叙事一句话能讲清吗？**

* “test-time compute 正在成为推理跃迁的核心，但社区缺少**可信**的 scaling 方法；我们把它变成一个可验证的统计选择问题，使 test-time scaling 不再依赖脆弱的 heuristics。”

### 5.3 创新点列表（novelty checklist，≥5；含“最接近 3 类工作”逐项对比）

1. **从‘marginal 校准’到‘selection-conditional 风险控制’**

* 对比：DS-CP 处理 domain shift 下 coverage，但不针对“从多个候选里挑一个”的选择偏差。([arXiv][7])
* 我们：对“最终被挑选输出”的错误率给 SCC 风格保证（用 PEMI 类选择性 conformal 思路）。([arXiv][68])

2. **把 SCOPE 类的“选择性可信 judge”迁移成“选择性可信 verifier 输出”**

* 对比：SCOPE 面向评测裁决，不处理 search/候选选择。([arXiv][10])
* 我们：面向推理系统输出，直接解决 reward hacking 在大 compute 下的核心痛点。

3. **统一 D1 的 test-time scaling 视角：Risk–Coverage–Compute 曲线**

* 对比：Best-of-N/InferenceTimePessimism 强调 scaling 单调性，但缺少跨 verifier/任务的统一风险报告。([ICML][2])

* 对比：VG-Search 研究验证粒度对效率/精度影响。([NeurIPS][3])

5. **可证伪预测：大 N 下 reward hacking 的‘转折点’由 verifier 的校准质量决定**

### 5.4 核心洞察（为什么不是工程堆砌，而是新视角/机制）

* **机制分解**：test-time compute 带来两股力量：

  1. “coverage 提升”——采样/搜索更可能碰到正确解；
  2. “选择放大误差”——verifier 的假阳性会随候选数增长被放大（reward hacking）。([ICML][2])
* 现有方法大多只在第 1 点做文章（堆 N / 改 search），或在第 2 点做启发式抑制（pessimism）。
* 我们的视角：**把第 2 点变成可控的统计风险预算**，从而让 compute scaling “可用、可报告、可复现”。

### 5.5 方法清晰定义（伪代码级别）——**SeCURE：Selective Conformal Uncertainty-Risk controlled sEarch**

#### 问题设定

* 给定 prompt (x)，生成器 (G) 产生候选 (y_1,\dots,y_N)。
* verifier (V) 给出分数或概率 (p_\phi(\text{correct}\mid x,y))。
* 目标：输出 (\hat y)，并满足：

  * **Risk constraint**：(\Pr(\hat y \text{ is wrong}) \le \alpha)（对“被输出的那个”）
  * 同时尽可能提高 accuracy / utility，并控制 compute（N、tokens、verifier calls）。

#### 版本 0（2–3 周最小闭环）：Bonferroni-split conformal（强但偏保守）

* 对每个候选单独做 conformal 分类预测集；
* 用 Bonferroni 处理“挑一个输出”的多重性：(\alpha'=\alpha/N)。

#### 版本 1（论文主贡献之一）：选择性 conformal（SCC）以减少保守性

* 使用 PEMI 类“选择事件保持”的校准方式，直接对“被挑选出的样本”保证 coverage/risk。([arXiv][68])

#### 推理伪代码（统一接口：可用于 Best-of-N / VG-Search / PRM-search）

```text
Inputs: prompt x, generator G, verifier V, calibration set D_cal,
        risk level alpha, max candidates N_max, selection rule S(·)

Precompute: conformal calibrator C from D_cal (split conformal),
            optionally selective calibrator C_sel for rule S (PEMI-style)

for j = 1..N_max:
    y_j ~ G(· | x)   # sample / expand node
    s_j = V(x, y_j)  # score/probability
    cert_j = C_sel.is_certified(x, y_j, s_j, alpha)  # or Bonferroni C
    
Collect all certified candidates Y_cert = {y_j : cert_j = True}
if Y_cert non-empty:
    return argmax_{y in Y_cert} Utility(y)   # e.g., verifier score or task-specific
else:
    return ABSTAIN or fallback (best score, but marked "uncertified")
```

### 5.6 训练/推理技术细节（损失、超参、复杂度、并行策略）

#### Verifier 训练（推荐：小中模型，避免堆算力）

* 数据：((x,y,label))；label 来自**可验证判定**：

  * math：最终答案匹配；
  * code：单元测试；
  * tool-use：环境状态/隐藏 DAG 终点（如可用）。
* 模型：7B/14B 级别 LM 做二分类头（LoRA），或专用 RM/PRM。
* 损失：标准 BCE / CE；可选加入 calibration-friendly 正则（如温度/label smoothing）。
* 关键超参（起步值）：

  * lr 1e-4（LoRA），batch 256–1024（按序列长度），epoch 1–3；
  * 正负样本比按任务平衡（避免 trivially negative）。

#### Conformal 校准

* Split：train verifier 用 D_train；校准用 D_cal（≥10k/任务）；测试用 D_test。
* 非一致性分数（示例）：(a(x,y,label)=1-p_\phi(label|x,y))。
* 版本 0：Bonferroni：(\alpha'=\alpha/N)（最稳，最简单）。
* 版本 1：选择性校准（PEMI）：按选择规则 S 构造 selection-preserving reference set，减少保守性。([arXiv][68])
* 版本 2（可选）：domain shift reweight（DS-CP）：基于 prompt embedding 相似度对 D_cal 加权。([arXiv][7])

#### 推理复杂度

* 主要成本：生成 N 个候选的 token 成本 + N 次 verifier 前向。
* 并行：

  * 候选生成在 16×80GB 上做批量 sampling；
  * verifier 批量打分（吞吐极高）。
* 相比训练：主要是推理期实验，**非常契合“非堆预训练规模”约束**。

### 5.8 预期会失败的设置（≥2；失败也有价值）

1. **极端稀有正确解任务**（coverage 太低）：

* 现象：认证通过率接近 0（几乎总 abstain），准确率提升有限。
* 价值：证明“风险控制≠万能”，需要与 coverage/生成策略协同（推动后续 work）。

2. **强分布漂移 + verifier 失真**（例如强对抗 prompt / 新领域）：

* 现象：若不做 DS-CP/选择性校准，认证风险会失控或过度保守。
* 价值：用系统实验量化 domain shift 与 selection bias 的交互；为方法边界提供清晰结论。([arXiv][7])

---

### 5.9 关键实验（Minimum viable experiments ≥6；每个都给可证伪预测）

> 统一输出：**Risk–Coverage–Compute 曲线** + accuracy/pass@k；并报告 bootstrap CI。

1. **复现并量化“大 N degrade”**（Best-of-N vs N）

* 任务：MATH / GSM8K / HumanEval（至少 2 类）。
* 预测：plain Best-of-N 随 N 增大出现性能回落（reward hacking/误差放大）。([ICML][2])

2. **SeCURE（Bonferroni 版）风险控制有效性**

* 指标：被“certified correct”的样本中实际错误率 ≤ α（如 5%）。
* 预测：经验错误率稳定低于阈值；若不校准则显著超标（可证伪）。

3. **SeCURE 缓解大 N degrade（准确率/效用）**

* 预测：在相同 N 下，SeCURE 的 accuracy 不会像 plain Best-of-N 那样回落；至少“回落点右移”，最好近似单调。

4. **选择偏差实验：固定候选池，改变 selection rule**

* 做法：同一批候选，比较：max-score / pessimism / random among certified。
* 预测：没有 selection-conditional 处理时，风险估计会随规则变化而漂移；SeCURE-PEMI 能稳定。([arXiv][68])

5. **验证粒度可组合性：接入 VG-Search/PRM-search**

* 对比：VG-Search 原版 vs 加入 risk-controlled certification。([NeurIPS][3])
* 预测：在相同 FLOPs/调用次数下，认证版本能降低“高分错解”的比例，并改善 compute efficiency。

6. **Domain shift stress test：风格/领域/对抗扰动**

* 对比：标准 conformal vs DS-CP reweight。([arXiv][7])
* 预测：DS-CP 在 shift 下保持更接近目标 risk/coverage；标准版本 under-cover 或过保守。

7. **机制验证：误差放大定律（Scaling/complexity lens）**

* 做法：估计 verifier 的 FPR（假阳性率）与候选数 N 的关系；验证“极值选择”导致错误概率随 N 上升。
* 预测：plain Best-of-N 的错误主要来自“高分假阳性尾部”；SeCURE 抑制该尾部。

---

### 5.10 风险与备选路线（Plan B/C）

**主要风险**

* R1：PEMI/选择性 conformal 落地复杂，可能工程代价高。([arXiv][68])
* R2：在某些任务上过于保守（coverage 低），导致收益有限。
* R3：verifier 训练数据偏差导致校准不稳。

**Plan B（更稳、更快发表）**

* 只做 **Bonferroni-split conformal + DS-CP**：

  * 依旧提供强保证与大规模实证；
  * 把 PEMI 作为 “advanced” 扩展章节。([arXiv][7])

**Plan C（更偏 NeurIPS best-paper 叙事）**

* 把 SeCURE 与 **InferenceTimePessimism** 结合：

  * “pessimism 给最优性/单调性”；
  * “SeCURE 给有限样本风险证书 + 可报告协议”。([ICML][2])

---

### 5.11 如果结果不如预期：诊断树（debug roadmap）

1. **先看风险是否失控**（certified 错误率 > α）

* 查：校准集是否泄漏/太小 → 增大 D_cal；
* 查：verifier 输出是否严重未校准 → 温度校准/重新训练；
* 查：selection bias（挑选规则）是否破坏假设 → 上 PEMI 或保守 Bonferroni。([arXiv][68])

2. **风险控制住了但 accuracy 不升（太保守）**

* 查 coverage：

  * coverage 低 → 提升生成 coverage（更好采样、轻微 finetune、或用 pass@N 友好训练；已有工作指出训练与 pass@N 会失配）([NeurIPS][1])
* 查 verifier 质量：

  * verifier AUC 低 → 换更强 verifier（RRMs）([OpenReview][4]) 或加过程信号（PRM）。

3. **只在 shift 下崩**

* 上 DS-CP reweight；或分域校准；或报告 shift-aware 风险曲线。([arXiv][7])

## 6. 执行计划（里程碑 + 资源估算 + 实验清单）

> 目标：**10 周**做出 NeurIPS 级完整闭环；其中 **前 2–3 周**必须拿到“是否值得继续”的强信号。

### 6.1 数据与基准选择（含理由）

**强推荐优先选择“可验证”任务**（减少 judge 争议，保证可复现）：

* **数学**：MATH（论文中常用；NeurIPS’25 工作也用）([NeurIPS][1])
* **代码**：HumanEval/MBPP（单元测试可验证）
* **工具调用/多步函数调用**：FuncBenchGen（hidden DAG、污染可控）([arXiv][69])
* **可选扩展**：

  * theorem proving / Lean：用于展示“验证器更强”的极端场景（但工程更重）。([NeurIPS][70])

### 6.2 周计划（按里程碑）

**Week 1：复现实验地基 + 基线跑通**

* 跑通：plain Best-of-N（N=1,4,8,16,32,64）+ pass@N 指标
* 复现：大 N degrade（至少在 1–2 个任务上出现）([ICML][2])
* 产出：第一版 Risk–Compute 曲线草图（哪怕没有风险证书）

**Week 2：Verifier 数据管线 + 训练一个“够用”的 verifier**

* 生成候选：用开源生成器采样（温度/Top-p 多设置），得到 (x,y,label)
* 训练 verifier（LoRA，小模型起步）
* 产出：verifier ROC/AUC、校准曲线（reliability diagram）

**Week 3（最小闭环）：SeCURE v0（Bonferroni conformal）上线**

* 实现 split conformal + Bonferroni 多重性
* 验证：certified 错误率 ≤ α（核心 gate）
* 若通过：进入 Week 4；若失败：走 debug tree（5.11）

**Week 4–5：系统化实验 + ablation**

* 任务扩展到 3 类（math+code+tool）
* Ablation：无校准/温度校准/DS-CP（shift）([arXiv][7])
* 对比强基线：InferenceTimePessimism、VG-Search、RRMs（可只做 1–2 个强基线深入）([ICML][2])

**Week 6：SeCURE v1（选择性 conformal / PEMI）**

* 实现 PEMI 风格 selection-conditional 校准，降低保守性([arXiv][68])
* 对比：coverage 提升多少、在同等风险下 compute 省多少

**Week 7–8：机制与理论章节打磨**

* 机制实验：误差放大定律、尾部假阳性分析
* 理论：给出 risk bound（Bonferroni 版）+ SCC 版（PEMI 版）的条件与结论([arXiv][68])

**Week 9–10：写作 + 复现包**

* 开源：代码、配置、数据生成脚本、随机种子、表格自动生成
* 最终：论文图表定稿 + 附录补齐统计显著性

### 6.3 评测协议（主指标 + 鲁棒性 + 显著性 + 机制验证）

* 主指标：accuracy / pass@k（按任务）
* 核心新增指标（必须）：

  * **Certified error rate**（被认证输出的错误率）
  * **Coverage**（认证通过率）
  * **Compute**（tokens、verifier calls、wall time）
  * 三者联合：**Risk–Coverage–Compute 曲线**
* 鲁棒性：prompt 风格 shift、对抗扰动、domain shift（配 DS-CP）([arXiv][7])
* 显著性：bootstrap CI；多 seed；报告方差
* 机制验证：tail analysis（高分错解比例随 N 变化）

### 6.4 资源估算（显存、训练时间、GPU 天数上限、并行策略）

* Verifier LoRA 训练：

  * 7B 级：16×80GB 下通常 **<1 天/多任务**（取决于序列长度与样本量）
* 候选生成：

  * 主要耗时在推理；可以并行跑 N 曲线（N=1..64），吞吐受限于生成器
* 总 GPU-days（保守上限）：

  * **≤ 200 GPU-days**（重点花在大规模推理曲线与多任务 ablation，而非大训练）

### 6.5 复现与工程组织

* 实验配置：Hydra/YAML（每个实验可复跑）
* 日志：W&B + 本地落盘（每个 run 记录数据版本、git hash、seed）
* 超参策略：

  * verifier 先小范围 grid（lr、epoch、正负比）
  * conformal 参数固定（α、cal size），避免“调参污染结论”

## 7. 写作框架（标题×5、摘要骨架、贡献点、图表草图、related work）

### 7.1 标题候选（5）

1. **SeCURE Search: Risk-Controlled Test-Time Scaling for Verifier-Guided LLM Reasoning**
2. **From Reward Hacking to Risk Budgets: Selection-Conditional Guarantees for Best-of-N and Beyond**
3. **Certified Test-Time Compute: Conformal Risk Control for Verifier-Guided Generation**
4. **Trustworthy Test-Time Scaling under Selection Bias and Domain Shift**
5. **Risk–Coverage–Compute: A Principled Framework for Reliable Inference-Time Alignment**

### 7.2 摘要骨架（可直接写进论文）

* 背景：test-time compute scaling 强，但 reward hacking/选择偏差使其不可靠。([ICML][2])
* 问题：在多候选选择、分布漂移下，如何对最终输出提供可复现的风险保证？([arXiv][7])
* 方法：SeCURE——把 verifier-guided search 形式化为选择性预测；用 conformal +（可选）PEMI 进行 selection-conditional 风险控制。([arXiv][68])
* 理论：给出有限样本风险上界（Bonferroni 版）与更不保守的 SCC 版（PEMI 版）。
* 实验：在 math/code/tool 的可验证基准上，SeCURE 在控制 certified error 的同时，显著缓解大 compute 下的 degrade，并给出 Risk–Coverage–Compute 曲线。
* 影响：为“可信 test-time scaling”提供通用组件与评测协议。

### 7.3 贡献点 bullet（best-paper 风格）

* **(Theory)** 首个面向 verifier-guided test-time search 的 **selection-conditional 风险控制**框架。
* **(Mechanism)** 解释并量化“选择放大 verifier 误差”导致 reward hacking 的机制，并用风险预算解耦 compute 与错误放大。([ICML][2])
* **(Evaluation)** 提出统一 **Risk–Coverage–Compute** 评测协议（+ CI），跨任务可复现。
* **(Systems)** 与 Best-of-N / VG-Search / PRM-search 可组合，形成可落地推理系统组件。([NeurIPS][3])

### 7.4 主要图表草图说明（建议至少 5 张）

1. **核心机制图**：coverage↑ vs error-amplification↑ 两股力量；SeCURE 用风险预算压住后者。
2. **Risk–Coverage–Compute 曲线**（主图）：不同方法的 Pareto front。
3. **Accuracy vs N 曲线**：plain Best-of-N 回落；SeCURE 稳定/单调。([ICML][2])
4. **选择偏差示意**：为什么“挑出来的样本”破坏 IID；PEMI 如何修复。([arXiv][68])
5. **Domain shift 图**：标准 conformal vs DS-CP 在 shift 下的 under/over-coverage 对比。([arXiv][7])

### 7.5 Related work 组织方式（按“统一原则”写）

* A. Inference-time alignment / test-time compute scaling（Best-of-N、pessimism、verification granularity）([ICML][2])
* B. Verifier / reward modeling with test-time compute（RRMs 等）([OpenReview][4])
* C. Uncertainty quantification & conformal prediction for LLMs（DS-CP、linguistic calibration、selective prediction）([arXiv][7])
* D. Selective/conditional inference for conformal（PEMI）([arXiv][68])
* E. LLM-as-judge statistical correction（作为动机与对照：为何需要 CI/纠偏）([arXiv][9])
